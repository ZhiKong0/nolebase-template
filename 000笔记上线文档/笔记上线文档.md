## 二、本地开发

### 2.1 环境要求

- **Node.js**: >= 18.0.0 (推荐 20.x)
- **包管理器**: npm / pnpm / yarn

### 2.2 安装依赖

```bash
# 克隆仓库
git clone https://github.com/ZhiKong0/nolebase-template.git
cd nolebase-template

# 安装依赖
npm install
```

### 2.3 常用命令

| 命令 | 作用 |
|------|------|
| `npm run dev` 或 `npm run docs:dev` | 启动开发服务器 (http://localhost:5173) |
| `npm run build` 或 `npm run docs:build` | 构建生产版本 |
| `npm run serve` 或 `npm run docs:serve` | 预览生产构建 |
| `npm run update` | 执行更新脚本 |

### 2.4 开发流程

```
1. 创建/编辑笔记文件 (.md)
   ↓
2. 本地预览 (npm run dev)
   ↓
3. 提交到 git
   ↓
4. 推送到 main 分支
   ↓
5. 自动部署到 GitHub Pages
```

## 三、部署配置详解

### 3.1 GitHub Actions 工作流

**文件**: `.github/workflows/deploy.yml`

```yaml
name: 部署到 GitHub Pages

on:
  push:
    branches: [main]      # main 分支推送时触发
  workflow_dispatch:      # 支持手动触发

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci        # 安装依赖
      - run: npm run docs:build  # 构建
      - uses: actions/upload-pages-artifact@v3
        with:
          path: .vitepress/dist   # 构建产物目录

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - uses: actions/deploy-pages@v4  # 部署到 GitHub Pages
```

### 3.2 关键配置说明

#### A. VitePress 配置 (`vite.config.ts`)

```typescript
// SSR 配置 - 确保这些包不会被外部化
ssr: {
  noExternal: [
    '@nolebase/vitepress-plugin-enhanced-readabilities',
    '@nolebase/vitepress-plugin-highlight-targeted-heading',
    '@nolebase/vitepress-plugin-inline-link-preview',
    '@unlazy/vue',  // 必须添加，否则构建失败
  ],
}
```

#### B. GitHub Pages 路径配置 (`.vitepress/config.ts`)

```typescript
export default defineConfig({
  base: '/nolebase-template/',  // 仓库名作为 base 路径
  // ...
})
```

**注意**: 如果仓库名更改，需要同步修改 `base` 配置。

## 四、上线步骤

### 4.1 首次部署

1. **Fork/创建仓库**
   ```bash
   # 如果是新项目，复制模板
   git clone https://github.com/ZhiKong0/nolebase-template.git my-notes
   cd my-notes
   git remote set-url origin https://github.com/YOUR_USERNAME/my-notes.git
   ```

2. **修改配置**
   - 更新 `metadata/index.ts` 中的站点信息
   - 更新 `.vitepress/config.ts` 中的 `base` 路径
   - 更新 `package.json` 中的作者信息

3. **启用 GitHub Pages**
   - 进入仓库 Settings → Pages
   - Source 选择 "GitHub Actions"

4. **推送触发部署**
   ```bash
   git add .
   git commit -m "Initial commit"
   git push origin main
   ```

5. **查看部署状态**
   - 进入仓库 Actions 标签页
   - 查看工作流运行状态
   - 绿色 ✓ 表示部署成功

### 4.2 更新内容

```bash
# 1. 编辑笔记文件
vim 笔记/xxx.md

# 2. 本地预览（可选）
npm run dev

# 3. 提交并推送
git add .
git commit -m "更新笔记"
git push origin main

# 4. 自动部署
# GitHub Actions 会自动执行部署流程
```

### 4.3 手动触发部署

如果需要强制重新部署：

1. 进入仓库 Actions 标签页
2. 选择 "部署到 GitHub Pages" 工作流
3. 点击 "Run workflow" 按钮

## 五、故障排查

### 5.1 构建失败

| 问题 | 原因 | 解决 |
|------|------|------|
| `Cannot find module '@unlazy/vue'` | SSR 外部化问题 | 确保 `@unlazy/vue` 在 `ssr.noExternal` 列表中 |
| `vue` 文件扩展名错误 | 构建工具问题 | 检查 `ssr.noExternal` 配置 |
| 路径 404 | `base` 配置错误 | 确认 `base` 与仓库名一致 |

### 5.2 部署失败

| 问题 | 原因 | 解决 |
|------|------|------|
| 权限不足 | `GITHUB_TOKEN` 权限 | 检查仓库 Settings → Actions → General → Workflow permissions |
| 部署未触发 | 分支名错误 | 确认推送到了 `main` 分支 |
