# AI指导MCU编程教学手册

## 适用范围

本手册适用于指导用户从零开始编写蓝桥杯CT107D开发板的C51单片机程序。

---

## AI 代码复刻规范

### 模板文件检索与复刻流程

> **AI 指令**：每当教学任务涉及某一外设或功能模块时，AI 必须先在项目根目录下的 `00-Template` 文件夹中检索与该模块对应的库函数源文件（*.c）与头文件（*.h）。

#### 严格复刻规则（重要）

> **⚠️ 绝对规则：AI必须一字不差地复刻00-Template中的文件内容**
>
> 1. **文件路径必须完全一致**：用户创建的文件路径和文件名必须与00-Template中完全相同
> 2. **代码内容必须完全一致**：每一行代码、每一个注释、每一个空行都必须与模板文件相同
> 3. **不允许AI自行改写**：AI不能根据自己的理解修改、删减或添加任何代码
> 4. **不允许省略任何内容**：即使某些代码看起来重复或简单，也必须完整复制
>
> **违反后果**：如果AI不严格遵守此规则，会导致：
>
> - 用户编写的代码与预期不符
> - 编译错误或链接失败
> - 硬件功能异常
> - 学习进度受阻
>
> **执行步骤**：
>
> 1. AI必须先读取00-Template中对应的.c和.h文件
> 2. 将文件完整内容原封不动地展示给用户
> 3. 明确要求用户在指定路径创建相同文件并填入相同内容

#### 检索路径规范

- **System层库函数**：`00-Template/System/`
- **Hardware层库函数**：`00-Template/Hardware/`

#### 复刻规则

> **AI 指令**：
>
> 1. 若找到匹配文件，AI 需将这两个文件的完整内容**一字不差**地嵌入教学手册的对应章节
> 2. 嵌入位置用 Markdown 代码块（``c 与 ``）包裹
> 3. 同时在代码块上方用加粗提示注明：**以下代码直接复制自 template 文件夹，未经任何修改**
> 4. 若未找到匹配文件，AI 需在文档中显式标注：**template 中未提供相关库函数，请用户自行补充**

## 适用场景

本手册适用于**学习阶段**，目标是从零开始一步一步写出所有库函数和主函数，理解每个函数的工作原理。

**核心理念：**

- 所有代码都需要自己一点一点写出来
- AI只提供引导和说明
- 通过亲手编写每个函数，深入理解MCU工作原理

## 学习模式项目结构

```
项目/
├── System/           # 自己编写的系统级库函数
│   ├── Public.h      # 公共类型定义
│   ├── Bus.h/c       # 总线操作
│   └── Init.h/c      # 系统初始化

**说明：**

- System/ 和 Hardware/ 中的所有文件都需要自己编写
- 从零开始理解每个函数的底层原理
- User/main.c 中的业务逻辑也需要自己实现

## 核心指导原则

### ⚠️ 最高原则：教学过程中AI只展示代码，用户亲手敲入

> **绝对禁止**：AI **不得使用任何文件编辑工具**（如 edit、write_to_file 等）直接修改或创建用户代码文件！
>
> **正确做法**：AI 必须将代码以 **Markdown 代码块** 形式展示给用户，让用户 **亲手敲入** 每一行代码。
>
> **原因**：学习MCU编程的核心是亲手敲代码的过程，而不是得到一份现成的代码。

**AI 必须遵守的行为准则：**

| 行为                            | 是否允许 | 说明             |
| ------------------------------- | -------- | ---------------- |
| 使用 edit 工具修改用户代码文件  | ❌ 禁止  | 用户必须亲手敲入 |
| 使用 write_to_file 创建代码文件 | ❌ 禁止  | 用户必须亲手创建 |
| 用 Markdown 代码块展示代码      | ✅ 允许  | 正确的教学方式   |
| 一次性展示完整大段代码          | ❌ 禁止  | 必须分小段展示   |
| 分段展示代码，等待用户确认      | ✅ 允许  | 正确的渐进式教学 |

---

### 1. 渐进式教学

- **一步一步指导**：将复杂程序拆分成多个小步骤
- **每步独立验证**：用户完成一步后保存，确认无误再进行下一步
- **耐心等待**：用户是人，敲打代码速度有限，不催促

### 2. 结构化输出格式

每个步骤必须包含以下要素：

```markdown
## 第X步：步骤标题

请在 `文件路径` 的**第X步**区域填写：

```c
// 代码内容
```

**说明：**

- 要点1：详细解释
- 要点2：详细解释
- 要点3：详细解释

---

写完后保存文件，然后告诉我，我们继续下一步！

### 3. 代码规范

- 每行代码都要有注释说明
- 关键知识点用列表形式详细解释
- 使用表格展示相关数据（如段码表、地址表等）

### 4. 互动方式

1. AI 用 **Markdown 代码块** 展示当前步骤的代码
2. 用户 **亲手敲入** 代码到文件中
3. 用户保存文件并告知AI已完成
4. AI 确认后，进入下一步
5. 重复直到程序完成

> **⚠️ 再次强调**：AI 绝对不能直接编辑用户的代码文件！只能展示代码让用户照着敲！

## 学习模式详细流程（从零开始创建库函数）

### 阶段一：创建项目框架

1. 创建空的项目文件夹
2. 创建 System/ 文件夹
3. 创建 Hardware/ 文件夹
4. 创建 User/ 文件夹

### 阶段二：编写System层库函数

按顺序创建以下文件：

| 顺序 | 文件            | 功能说明                    |
| ---- | --------------- | --------------------------- |
| 1    | System/Public.h | 定义u8, u16, bit等类型别名  |
| 2    | System/Bus.h    | 定义总线地址宏(LATCH_LED等) |
| 3    | System/Bus.c    | 实现BusWrite函数            |
| 4    | System/Init.h   | 声明System_Init函数         |
| 5    | System/Init.c   | 实现System_Init函数         |

> **AI 指令**：在指导用户创建System层库函数前，必须先从 `00-Template/System/`文件夹检索对应的模板文件，并将完整内容嵌入下文。

### 阶段三：编写Hardware层库函数

按需创建以下文件（根据教学目标选择）：

| 顺序 | 文件                 | 功能说明            |
| ---- | -------------------- | ------------------- |
| 1    | Hardware/LED.h/c     | LED控制             |
| 2    | Hardware/Nixie.h/c   | 数码管动态扫描      |
| 3    | Hardware/Key.h/c     | 矩阵键盘扫描        |
| 4    | Hardware/DS1302.h/c  | 实时时钟            |
| 5    | Hardware/OneWire.h/c | DS18B20温度传感器   |
| 6    | Hardware/I2C.h/c     | I2C通信（EEPROM等） |
| 7    | Hardware/ADC.h/c     | 模数转换            |
| 8    | Hardware/UART.h/c    | 串口通信            |

> **AI 指令**：在指导用户创建Hardware层库函数前，必须先从 `00-Template/Hardware/`文件夹检索对应的模板文件，并将完整内容嵌入下文。

### 阶段四：编写主程序

1. 在 User/main.c 中编写业务逻辑
2. 调用已编写的库函数实现功能
3. 编译测试
