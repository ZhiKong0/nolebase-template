## 一、函数总览

赛题BSP（`onewire.c`）提供的**4个核心函数**：

| 函数名 | 功能 | 调用层级 |
|--------|------|----------|
| `Delay_OneWire` | 底层：单总线延时 | 最底层 |
| `init_ds18b20` | 底层：初始化/复位 | 调用延时 |
| `Write_DS18B20` | 底层：写入1字节 | 调用延时 |
| `Read_DS18B20` | 底层：读取1字节 | 调用延时 |

**调用关系**：
```
用户代码
   ↓ 调用
DS18B20_ReadRawTemp()
   ↓ 内部调用
init_ds18b20() / Write_DS18B20() / Read_DS18B20()
   ↓ 底层调用
Delay_OneWire()
   ↓ 底层延时
for循环延时
```

## 三、Delay_OneWire 延时函数解析

### 3.1 完整源码

```c
/**
 * @brief 单总线延时函数
 * @param t 延时单位数
 * 
 * 延时原理：嵌套循环实现软件延时
 * - 外层while(t--)循环t次
 * - 内层for循环12次（约12个机器周期）
 * 
 * 延时计算（11.0592MHz晶振）：
 * - 机器周期 = 12 / 11.0592MHz ≈ 1.085μs
 * - for循环12次 ≈ 12 * 1.085μs ≈ 13μs
 * - while循环开销 ≈ 1μs
 * - 总延时 ≈ t * 14μs
 * 
 * 常用延时值：
 * - t=5: 约70μs（写时隙）
 * - t=10: 约140μs（读时隙采样等待）
 * - t=12: 约168μs（初始化前稳定）
 * - t=80: 约1120μs（复位脉冲）
 * - t=200: 约2.8ms（温度转换等待）
 */
void Delay_OneWire(unsigned int t)
{
	unsigned char i;
	while(t--){
		for(i=0;i<12;i++);
	}
}
```

### 3.2 延时时间计算

| 晶振频率 | 机器周期 | 内层循环12次 | 外层开销 | 单位t延时 |
|:--------:|:--------:|:------------:|:--------:|:---------:|
| 11.0592MHz | 1.085μs | ~13μs | ~1μs | **~14μs** |
| 12MHz | 1μs | ~12μs | ~1μs | **~13μs** |

**常用延时值对应实际时间**：

| 参数t | 实际延时 | 使用场景 |
|:-----:|:--------:|:--------:|
| 5 | ~70μs | 写时隙保持时间 |
| 10 | ~140μs | 读时隙采样等待 |
| 12 | ~168μs | 初始化前稳定 |
| 80 | ~1120μs | 复位脉冲（>480μs）|
| 200 | ~2.8ms | 温度转换等待 |

## 四、init_ds18b20 初始化/复位函数解析

### 4.1 完整源码

```c
/**
 * @brief DS18B20初始化（复位脉冲）
 * @retval 0: 初始化失败（无响应），1: 初始化成功
 * 
 * 单总线初始化时序：
 * 1. DQ=1：确保总线初始为高
 * 2. Delay(12)：延时等待总线稳定
 * 3. DQ=0：主机拉低DQ（启动复位脉冲）
 * 4. Delay(80)：延时约1120μs（复位脉冲宽度，>480μs）
 * 5. DQ=1：主机释放，上拉电阻拉高
 * 6. Delay(10)：延时约140μs（等待从机应答）
 * 7. initflag = DQ：采样DQ状态（0=有应答，1=无应答）
 * 8. Delay(5)：延时约70μs（等待从机释放总线）
 * 9. return initflag：返回初始化结果
 */
bit init_ds18b20(void)
{
  	bit initflag = 0;
  	
  	DQ = 1;
  	Delay_OneWire(12);    // 延时约168μs
  	DQ = 0;
  	Delay_OneWire(80);    // 延时约1120μs（复位脉冲>480μs）
  	DQ = 1;
  	Delay_OneWire(10);    // 延时约140μs（等待从机应答）
    initflag = DQ;        // 采样：0=有应答，1=无应答
  	Delay_OneWire(5);     // 延时约70μs（等待从机释放）
  
  	return initflag;      // 返回初始化结果
}
```

### 4.2 时序图详解

```
完整初始化时序：

DQ ─────┐         ┌────────────────────────┐         ┌───────┐     ┌────
        │ 168μs   │       1120μs           │  140μs  │60-240μs│70μs│
        └─────────┘                        └─────────┘        └─────┘
        高       低（复位脉冲）               高        低      高
                 主机拉低                    上拉高   从机拉低 从机释放
                                                      应答脉冲

关键时间点：
T0: DQ=1（初始高）
T1: Delay(12)后，DQ=0（拉低）
T2: Delay(80)后，DQ=1（释放）
T3: Delay(10)后，采样DQ（判断是否有从机应答）
T4: initflag = DQ
T5: Delay(5)后，函数返回
```

### 4.3 应答脉冲检测原理

```
DS18B20应答时序：

主机复位脉冲          应答脉冲              从机释放
    │                   │                    │
    ▼                   ▼                    ▼
DQ ─┐    ┌───────────────┐   ┌──────────────────
    └────┘               └───┘
    低电平>480μs         低电平60-240μs       高电平
    主机发送             从机发送              总线空闲

采样时机：
- 主机在释放DQ后60-240μs内采样
- 如果DQ=0：检测到从机应答（存在）
- 如果DQ=1：未检测到应答（不存在或故障）
```

### 4.4 关键代码解析

| 代码 | 作用 | 时序阶段 |
|------|------|----------|
| `DQ = 1` | 确保总线高电平 | 初始化准备 |
| `Delay(12)` | 延时168μs | 等待总线稳定 |
| `DQ = 0` | 拉低DQ | 启动复位脉冲 |
| `Delay(80)` | 延时1120μs | 复位脉冲宽度（必须>480μs）|
| `DQ = 1` | 释放DQ | 上拉电阻拉高，等待应答 |
| `Delay(10)` | 延时140μs | 等待从机拉低应答 |
| `initflag = DQ` | 采样总线 | 判断是否有设备应答 |
| `Delay(5)` | 延时70μs | 等待从机释放总线 |

## 五、Write_DS18B20 写数据函数解析

### 5.1 完整源码

```c
/**
 * @brief 向DS18B20写入一个字节
 * @param dat 要写入的8位数据
 * 
 * 写时隙时序（单总线协议）：
 * - 写"1"：主机拉低<15μs即释放
 * - 写"0"：主机拉低保持60-120μs
 * 
 * 本函数实现：
 * - 循环8次，每次写1位
 * - 数据从低位到高位发送（LSB FIRST）
 * - 使用Delay(5)保持时隙约70μs
 */
void Write_DS18B20(unsigned char dat)
{
	unsigned char i;
	for(i=0;i<8;i++)           // 循环8次，每次写1位
	{
		DQ = 0;                // 1. 拉低DQ，启动写时隙
		DQ = dat&0x01;         // 2. 放置数据（最低位）
		Delay_OneWire(5);      // 3. 延时约70μs（保持时隙）
		DQ = 1;                // 4. 释放DQ，结束写时隙
		dat >>= 1;             // 5. 数据右移，准备下一位
	}
	Delay_OneWire(5);          // 6. 额外延时
}
```

### 5.2 写时隙详解

```
写"1"时隙（DQ=1）：
DQ ────┐      ┌────────────────────────────
       │ 15us │
       └──────┘
       主机拉低 <15μs即释放，表示写"1"

写"0"时隙（DQ=0）：
DQ ────┐                                    ┌────
       │              60-120us              │
       └───────────────────────────────────┘
       主机拉低保持60μs以上，表示写"0"

本函数实现（统一使用70μs）：
DQ ────┐                                    ┌────
       │                70us                │
       └───────────────────────────────────┘
       延时5单位 ≈ 70μs，无论写0或1都保持70μs
       （兼容写0和写1，写1时稍慢但安全）
```

### 5.3 位发送过程示例

**写入 0xCC = 1100 1100（Skip ROM命令）**：

```
初始：dat = 0xCC = 1100 1100

bit0（LSB）= 0：
  DQ=0, DQ=dat&0x01=0, Delay(5), DQ=1, dat>>=1 → dat=0110 0110
  
bit1 = 0：
  DQ=0, DQ=0, Delay(5), DQ=1, dat>>=1 → dat=0011 0011
  
bit2 = 1：
  DQ=0, DQ=1, Delay(5), DQ=1, dat>>=1 → dat=0001 1001
  
bit3 = 1：
  DQ=0, DQ=1, Delay(5), DQ=1, dat>>=1 → dat=0000 1100
  
bit4 = 0：
  DQ=0, DQ=0, Delay(5), DQ=1, dat>>=1 → dat=0000 0110
  
bit5 = 0：
  DQ=0, DQ=0, Delay(5), DQ=1, dat>>=1 → dat=0000 0011
  
bit6 = 1：
  DQ=0, DQ=1, Delay(5), DQ=1, dat>>=1 → dat=0000 0001
  
bit7（MSB）= 1：
  DQ=0, DQ=1, Delay(5), DQ=1, dat>>=1 → dat=0000 0000

发送顺序：0→0→1→1→0→0→1→1（从低位到高位）
DS18B20接收顺序：bit0, bit1, bit2, ... , bit7
```

### 5.4 关键代码解析

| 代码 | 作用 | 说明 |
|------|------|------|
| `DQ = 0` | 启动写时隙 | 主机拉低DQ |
| `DQ = dat&0x01` | 放置数据位 | 取dat最低位 |
| `Delay(5)` | 保持时隙 | 约70μs |
| `DQ = 1` | 结束写时隙 | 释放总线 |
| `dat >>= 1` | 右移数据 | 准备下一位 |

## 六、Read_DS18B20 读数据函数解析

### 6.1 完整源码

```c
/**
 * @brief 从DS18B20读取一个字节
 * @retval 读取到的8位数据
 * 
 * 读时隙时序（单总线协议）：
 * 1. 主机拉低DQ >1μs启动读时隙
 * 2. 主机释放DQ（上拉电阻拉高）
 * 3. 从机在释放后输出数据到DQ
 * 4. 主机在15μs内采样DQ电平
 * 5. 读"1"：DQ为高；读"0"：DQ为低
 * 
 * 本函数实现：
 * - 循环8次，每次读1位
 * - 使用右移+或操作组装字节
 * - 数据从低位到高位接收（LSB FIRST）
 */
unsigned char Read_DS18B20(void)
{
	unsigned char i;
	unsigned char dat = 0;     // 初始化结果变量
  
	for(i=0;i<8;i++)           // 循环8次，每次读1位
	{
		DQ = 0;                // 1. 拉低DQ，启动读时隙
		dat >>= 1;             // 2. 结果右移，腾出最高位
		DQ = 1;                // 3. 释放DQ，从机输出数据
		if(DQ)                 // 4. 采样DQ电平
		{
			dat |= 0x80;       // 5. 如果为高，最高位置1
		}	    
		Delay_OneWire(5);      // 6. 延时完成时隙
	}
	return dat;                // 返回读取的字节
}
```

### 6.2 读时隙详解

```
读时隙时序：

DQ ────┐     ┌──────────────────────┐     ┌────
       │1us │  从机输出数据        │
       └───┘                      └─────┘
           ↑
        主机采样（15μs内）

详细时序：
1. 主机拉低1μs启动时隙
2. 主机立即释放（DQ=1）
3. 从机在释放后输出数据
4. 主机在15μs内采样DQ
   - DQ=1：读到"1"
   - DQ=0：读到"0"
5. 等待至60μs时隙结束
```

### 6.3 数据组装原理

```
数据组装过程（以接收0xAA = 1010 1010为例）：

初始: dat = 0000 0000

读bit0=0：
  dat >>= 1 → 0000 0000 (右移)
  DQ=0 → dat不变 → 0000 0000

读bit1=1：
  dat >>= 1 → 0000 0000 (右移)
  DQ=1 → dat |= 0x80 → 1000 0000 (0x80)

读bit2=0：
  dat >>= 1 → 0100 0000 (0x40)
  DQ=0 → dat不变

读bit3=1：
  dat >>= 1 → 0010 0000 (0x20)
  DQ=1 → dat |= 0x80 → 1010 0000 (0xA0)

读bit4=0：
  dat >>= 1 → 0101 0000 (0x50)
  DQ=0 → dat不变

读bit5=1：
  dat >>= 1 → 0010 1000 (0x28)
  DQ=1 → dat |= 0x80 → 1010 1000 (0xA8)

读bit6=0：
  dat >>= 1 → 0101 0100 (0x54)
  DQ=0 → dat不变

读bit7=1：
  dat >>= 1 → 0010 1010 (0x2A)
  DQ=1 → dat |= 0x80 → 1010 1010 (0xAA) ✓

最终结果：dat = 0xAA
```

### 6.4 为什么用 `dat >>= 1` 和 `dat |= 0x80`？

```
算法原理：
- 每次读取新bit前，先将dat右移1位（腾出最高位）
- 如果DQ=1，将最高位(bit7)置1
- 如果DQ=0，最高位保持0
- 8次循环后，8个bit按LSB FIRST顺序装入dat

LSB FIRST顺序装入的奇迹：
bit0装入bit7 → 右移 → 到bit6
bit1装入bit7 → 右移 → 到bit7, bit6到bit5
...
bit7装入bit7 → 不动

最终：bit0~bit7分别位于dat的bit0~bit7位置，顺序正确！
```

### 6.5 关键代码解析

| 代码 | 作用 | 说明 |
|------|------|------|
| `DQ = 0` | 启动读时隙 | 主机拉低DQ |
| `dat >>= 1` | 右移结果 | 腾出最高位给新数据 |
| `DQ = 1` | 释放总线 | 从机输出数据 |
| `if(DQ)` | 采样数据 | 读取DQ电平 |
| `dat \|= 0x80` | 置最高位 | 如果DQ=1，bit7=1 |
| `Delay(5)` | 完成时隙 | 等待60μs结束 |

## 八、常见错误与调试

### 8.1 初始化失败（返回1）

**可能原因**：
- DQ引脚未正确连接（检查P1.4）
- 延时太短（复位脉冲<480μs）
- 无外部上拉电阻（CT107D已内置）
- DS18B20损坏或未接入

**调试方法**：
```c
// 加LED指示
bit init_ds18b20_debug(void)
{
    bit flag = init_ds18b20();
    if(flag) LED_On();    // 失败亮灯
    else LED_Off();       // 成功灭灯
    return flag;
}
```

### 8.2 读取数据全为0或全为1

**可能原因**：
- 读时隙延时不对（采样时机错误）
- DQ引脚配置错误（未设为双向）
- 时隙间隔太短（<1μs恢复时间）

### 8.3 温度一直显示85

**可能原因**：
- 未执行温度转换命令（0x44）
- 转换延时不够（<750ms）
- 未执行首次空读清除上电默认值

**解决方案**：
```c
void main()
{
    System_Init();
    
    // 必须！上电首次读取丢弃85
    DS18B20_ReadRawTemp();
    
    while(1)
    {
        temp = DS18B20_ReadTemp_X(0);
        Display(temp);
    }
}
```
