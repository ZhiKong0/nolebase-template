# CT107D 总线工作原理详解

## 一、核心芯片概述

### 1.1 74HC138（3-8线译码器）

| 特性       | 说明                             |
| -------- | ------------------------------ |
| **核心功能** | **选择**哪个外设（开关/阀门）              |
| **比喻**   | 像**分岔路口的阀门**，决定水流向哪条管道         |
| **输入**   | 3位地址码 (CBA = P2.6 P2.5 P2.4)   |
| **输出**   | 8路选通信号 (Y0-Y7)，低电平有效           |
| **使能条件** | P2.7=0（WR低电平有效），G1=VCC，G2B=GND |
| **数量**   | 1片 (U17)                       |

**一句话描述：74HC138是"地址翻译官"——把P2口的二进制地址翻译成"打开哪扇门"**

### 1.2 74HC573（8位锁存器）

| 特性               | 说明                                                                                   |
| ------------------ | -------------------------------------------------------------------------------------- |
| **核心功能** | **保持**外设状态（记忆/锁存）                                                    |
| **比喻**     | 像**水桶**，装水后保持水量不变；也像**单向门**，关上门后外面变化不影响里面 |
| **输入**     | 8位数据 (D0-D7 = P0.0-P0.7)                                                            |
| **输出**     | 8位保持数据 (Q0-Q7)                                                                    |
| **控制端**   | LE (Latch Enable)：高电平=直通，低电平=锁存                                            |
| **数量**     | 4片 (U1, U2, U3, U6)                                                                   |

**一句话描述：74HC573是"记忆保持器"——记住P0口的数据，即使P0口后续变化也不影响外设**

## 三、74HC138 译码器详解

### 3.1 真值表

| P2值 | 二进制    | CBA | Y输出        | 选中的外设            | 用途                   |
| ---- | --------- | --- | ------------ | --------------------- | ---------------------- |
| 0x80 | 1000 0000 | 100 | **Y4** | LED锁存器 (U3)        | 控制8个LED             |
| 0xA0 | 1010 0000 | 101 | **Y5** | ULN锁存器 (U6)        | 控制蜂鸣器/继电器      |
| 0xC0 | 1100 0000 | 110 | **Y6** | 数码管位选锁存器 (U2) | 选通数码管位           |
| 0xE0 | 1110 0000 | 111 | **Y7** | 数码管段选锁存器 (U1) | 控制数码管段           |
| 0x00 | 0000 0000 | 000 | 无           | 关闭所有锁存器        | **必须的操作！** |

## 五、总线操作的核心：为什么必须"关门"？

### 5.1 错误示范（数据污染）

```c
// ❌ 错误写法：不关Y4就操作Y5
void Wrong_Control(void)
{
    // [步骤1] 打开Y4通道，让LED全亮
    P2 = 0x80;      // Y4选通，LED锁存器的LE=高电平（直通模式）
    P0 = 0x00;      // LED全亮 (低电平亮)
  
    // [错误] 没有执行 P2 = 0x00 关闭Y4！
    // 此时LED锁存器的LE仍然是高电平，处于"直通模式"
  
    // [步骤2] 打开Y5通道，想控制蜂鸣器
    P2 = 0xA0;      // Y5选通
    P0 = 0x40;      // 意图：蜂鸣器响
    // 但是！！由于Y4没关，P0=0x40也传到了LED锁存器！
    // LED状态变成了 0x40 (0100 0000)，大部分熄灭！
}
```

**问题本质**：74HC573的LE=高电平时是**直通模式**，P0的任何变化会实时反映到输出端。

### 5.2 正确示范（先关门）

```c
// ✅ 正确写法：操作完一个外设必须关门
void BusWrite(u8 addr, u8 dat)
{
    P2 = addr;      // [1] 选通对应锁存器，LE=高电平
    P0 = dat;       // [2] 输出数据
    _nop_();        // [3] 短暂延时，确保数据稳定
    P2 = 0x00;      // [4] 关闭锁存器，LE=低电平，数据被"锁存"住
}

void Correct_Control(void)
{
    // 控制LED全亮
    BusWrite(0x80, 0x00);   // Y4通道，LED全亮，关门后LED状态被保持
  
    // 控制蜂鸣器
    BusWrite(0xA0, 0x40);   // Y5通道，蜂鸣器响，不影响LED！
  
    // LED保持全亮，不受后续P0操作影响 ✓
}
```

## 六、总线操作四步法则

### 6.1 必须遵循的时序

```
┌────────────────────────────────────────────────────────────┐
│                   正确的总线写入时序                         │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  步骤1: 选通                                                │
│    P2 = addr;    // 选通对应锁存器，LE=高电平               │
│                  // 74HC573进入"直通模式"                    │
│                                                            │
│  步骤2: 写入                                                │
│    P0 = dat;     // 数据输出到P0口                          │
│                  // 由于LE=高，数据直通到Q端                │
│                                                            │
│  步骤3: 延时                                                │
│    _nop_();      // 短暂延时(约1us)                         │
│                  // 确保数据稳定建立                        │
│                                                            │
│  步骤4: 关闭（关键！）                                       │
│    P2 = 0x00;    // 关闭锁存器，LE=低电平                   │
│                  // 74HC573进入"锁存模式"                   │
│                  // Q端保持最后状态，不再跟随D端            │
│                                                            │
│  关键：必须执行步骤4！否则后续P0操作会"污染"该外设         │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 6.2 直观比喻

想象你在给不同的房间（外设）送快递（数据）：

- **P2口** = 房间门牌号选择器
- **P0口** = 传送带，所有房间共享
- **74HC573的LE** = 房间门
  - LE=高 = 门开着（直通），传送带上的东西会直接掉进去
  - LE=低 = 门关着（锁存），之前放进去的东西被保存，不受传送带影响

**错误操作**：去LED房间送完货不关门，然后去蜂鸣器房间送货，结果传送带上的货也掉进了LED房间！

**正确操作**：每送完一个房间，必须先关门（P2=0x00），再去下一个房间。

## 七、常见错误与排查

### 7.1 现象与原因对照

| 现象         | 可能原因                  |
| ---------- | --------------------- |
| LED乱跳/不受控制 | 操作其他外设后没有 `P2=0x00`关门 |
| 数码管有残影/重影  | 动态扫描时没有在切换位选前消影       |
| 蜂鸣器不响      | 地址写错（写成了0x80而不是0xA0）  |
| LED状态反了    | 逻辑搞反，CT107D是**低电平亮**  |
| 所有外设都不工作   | 跳线帽没接好（J2/J5等）        |

### 7.2 调试建议

```c
// 在BusWrite中加入调试输出（仅在调试时用）

{
    P2 = addr;      // 选通

    P2 = 0x00;      // 关门 - 这行必须有！
  
    // 调试时可加延时观察
    // Delay_ms(100);
}
```

## 九、总结

### 核心要点

1. **74HC138是"选择器"**：负责根据P2口的地址，选通对应的74HC573锁存器
2. **74HC573是"保持器"**：负责记住P0口的数据并保持输出给外设
3. **必须"关门"**：操作完任何外设后，必须执行 `P2=0x00`关闭锁存器，否则后续P0操作会污染该外设
4. **四步操作法**：选通→写入→延时→关闭，缺一不可

### 记忆口诀

> **138选通道，573保状态，操作完要关门，否则互相害！**
