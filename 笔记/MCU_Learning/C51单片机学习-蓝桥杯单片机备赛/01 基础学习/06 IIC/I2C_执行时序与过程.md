# I2C 执行时序图与执行过程

## 一、I2C起始条件时序

```
SDA  ────┐      ┌──────
         └──────┘
SCL  ────────────┐
                 └──────
                  ↑
              起始条件
                  
时序：SDA=1, SCL=1 -> 延时 -> SDA=0 (SCL高时SDA下降沿)
```

**关键点**：
- SCL高电平期间，SDA从高变低
- 表示通信开始，主机占用总线

---

## 二、I2C停止条件时序

```
SDA  ────┐      ┌──────
         └──────┘
SCL  ────────────┐
                 └──────
                  ↑
              停止条件
                  
时序：SDA=0, SCL=1 -> 延时 -> SDA=1 (SCL高时SDA上升沿)
```

**关键点**：
- SCL高电平期间，SDA从低变高
- 表示通信结束，释放总线

---

## 三、I2C数据位传输时序

```
SCL  ───┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──
        └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘
        ↑  ↑  ↑  ↑  ↑  ↑  ↑  ↑
        D7 D6 D5 D4 D3 D2 D1 D0 (MSB先发)
        
SDA  ───┬──┐  ┌──┐  ┌──┐
           └──┘  └──┘  └─────────...
           
时序：SCL低 -> 放数据 -> SCL高 -> 延时 -> SCL低
```

**关键点**：
- SCL低电平时改变SDA数据
- SCL高电平时数据稳定，从机采样
- 高位先发（MSB First）

---

## 四、I2C应答信号时序

```
发送方释放SDA:
SCL  ───┐  ┌──┐
        └──┘  └──┘
SDA  ───┐      ┌────
        └──────┘
        发送方释放

接收方拉低SDA(ACK):
SCL  ───┐  ┌──┐
        └──┘  └──┘
SDA  ───┐  ┌──┐
        └──┘  └────
        接收方拉低=ACK

或接收方保持高(NACK):
SCL  ───┐  ┌──┐
        └──┘  └──┘
SDA  ─────────────────
        保持高=NACK
```

**关键点**：
- 发送方发送完8位后释放SDA
- 接收方在第9个时钟周期拉低SDA表示ACK
- 保持高电平表示NACK

## 五、函数一：I2C_Start - 启动信号

### 代码实现

```c
void I2C_Start(void)
{
    SDA = 1;           // SDA先置高
    SCL = 1;           // SCL置高
    I2C_Delay();       // 延时，确保信号稳定
    SDA = 0;           // SCL高时SDA下降沿 = 启动信号
    I2C_Delay();       // 延时
    SCL = 0;           // SCL拉低，准备发送数据
}
```

### 执行过程示例

```
调用：I2C_Start()

初始状态：SDA=?, SCL=?

步骤1：SDA = 1
    SDA线被拉至高电平
    
步骤2：SCL = 1
    SCL线被拉至高电平
    此时：SDA=1, SCL=1 (空闲状态)
    
步骤3：I2C_Delay()
    延时约5μs，确保信号稳定
    
步骤4：SDA = 0
    在SCL保持高电平时，SDA变为低电平
    产生下降沿，这就是起始条件！
    
步骤5：I2C_Delay()
    延时约5μs
    
步骤6：SCL = 0
    SCL拉低，准备开始数据传输
    
最终状态：SDA=0, SCL=0，总线被主机占用，可以开始传输数据

时序图解：
SDA:  ────┐          ┌──────
         └──────────┘
         ↑起始位（SCL高时SDA下降沿）
SCL:  ───────────────┐
                     └──────
```

## 六、函数二：I2C_Stop - 停止信号

### 代码实现

```c
void I2C_Stop(void)
{
    SDA = 0;           // SDA先置低
    SCL = 1;           // SCL置高
    I2C_Delay();       // 延时
    SDA = 1;           // SCL高时SDA上升沿 = 停止信号
    I2C_Delay();       // 延时
}
```

### 执行过程示例

```
调用：I2C_Stop()

步骤1：SDA = 0
    确保SDA为低电平
    
步骤2：SCL = 1
    SCL置高
    此时：SDA=0, SCL=1
    
步骤3：I2C_Delay()
    延时约5μs
    
步骤4：SDA = 1
    在SCL保持高电平时，SDA变为高电平
    产生上升沿，这就是停止条件！
    
步骤5：I2C_Delay()
    延时约5μs
    
最终状态：SDA=1, SCL=1，总线释放，回到空闲状态

时序图解：
SDA:  ────┐          ┌──────
         └──────────┘
                  ↑停止位（SCL高时SDA上升沿）
SCL:  ────────────┐
                  └──────
```

## 七、函数三：I2C_SendByte - 发送一个字节

### 代码实现

```c
void I2C_SendByte(u8 dat)
{
    u8 i;
    for(i = 0; i < 8; i++)          // 循环8次，发送8位
    {
        SCL = 0;                     // SCL低电平，准备改变SDA
        if(dat & 0x80)               // 判断最高位(MSB)
            SDA = 1;
        else
            SDA = 0;
        I2C_Delay();                 // 延时
        SCL = 1;                     // SCL高电平，数据稳定
        I2C_Delay();                 // 延时，从机读取数据
        dat <<= 1;                   // 左移1位，准备发送下一位
    }
    SCL = 0;                         // SCL拉低，准备接收应答
}
```

### 执行过程示例：发送 `0x55`（二进制 01010101）

```
调用：I2C_SendByte(0x55)

初始：dat = 0x55 = 0101 0101（二进制）

第1轮（i=0，发送bit7=0）：
    SCL = 0                    // 时钟拉低
    dat & 0x80 = 01010101 & 10000000 = 0 → SDA = 0
    I2C_Delay()                // 延时
    SCL = 1                    // 时钟拉高，从机采样SDA=0
    I2C_Delay()                // 延时
    dat <<= 1 → dat = 1010 1010

第2轮（i=1，发送bit6=1）：
    SCL = 0
    dat & 0x80 = 10101010 & 10000000 = 1 → SDA = 1
    I2C_Delay()
    SCL = 1                    // 从机采样SDA=1
    I2C_Delay()
    dat <<= 1 → dat = 0101 0100

第3轮（i=2，发送bit5=0）：
    SCL = 0
    dat & 0x80 = 0 → SDA = 0
    I2C_Delay()
    SCL = 1                    // 从机采样SDA=0
    I2C_Delay()
    dat <<= 1 → dat = 1010 1000

第4轮（i=3，发送bit4=1）：
    SCL = 0
    dat & 0x80 = 1 → SDA = 1
    I2C_Delay()
    SCL = 1                    // 从机采样SDA=1
    I2C_Delay()
    dat <<= 1 → dat = 0101 0000

第5轮（i=4，发送bit3=0）：
    SCL = 0
    dat & 0x80 = 0 → SDA = 0
    I2C_Delay()
    SCL = 1                    // 从机采样SDA=0
    I2C_Delay()
    dat <<= 1 → dat = 1010 0000

第6轮（i=5，发送bit2=1）：
    SCL = 0
    dat & 0x80 = 1 → SDA = 1
    I2C_Delay()
    SCL = 1                    // 从机采样SDA=1
    I2C_Delay()
    dat <<= 1 → dat = 0100 0000

第7轮（i=6，发送bit1=0）：
    SCL = 0
    dat & 0x80 = 0 → SDA = 0
    I2C_Delay()
    SCL = 1                    // 从机采样SDA=0
    I2C_Delay()
    dat <<= 1 → dat = 1000 0000

第8轮（i=7，发送bit0=1）：
    SCL = 0
    dat & 0x80 = 1 → SDA = 1
    I2C_Delay()
    SCL = 1                    // 从机采样SDA=1
    I2C_Delay()
    dat <<= 1 → dat = 0000 0000

SCL = 0                        // 最后SCL拉低

发送顺序：0→1→0→1→0→1→0→1（从高位到低位）
对应：bit7, bit6, bit5, bit4, bit3, bit2, bit1, bit0

时序图解：
SCL:  ──┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
        └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘
        D7  D6  D5  D4  D3  D2  D1  D0
SDA:  ────┐   ┌───┐   ┌───┐   ┌───┐
            └───┘   └───┘   └───┘   └───
            0   1   0   1   0   1   0   1
```

## 八、函数四：I2C_RecvByte - 接收一个字节

### 代码实现

```c
u8 I2C_RecvByte(void)
{
    u8 i, dat = 0;
    SDA = 1;                         // 释放SDA，由从机驱动
    for(i = 0; i < 8; i++)
    {
        SCL = 1;                     // SCL高电平，数据稳定
        I2C_Delay();                 // 延时
        dat <<= 1;                   // 左移1位，腾出最低位
        if(SDA)                      // 采样SDA
            dat |= 0x01;             // SDA=1，最低位置1
        SCL = 0;                     // SCL低电平，准备下一位
        I2C_Delay();                 // 延时
    }
    return dat;
}
```

### 执行过程示例：接收 `0x55`（从机发送 01010101）

```
调用：I2C_RecvByte()

初始：dat = 0x00 = 0000 0000
SDA = 1（释放SDA，由从机驱动）

假设从机要发送 0x55 = 0101 0101（从高位到低位）

第1轮（i=0，接收bit7=0）：
    SCL = 1                    // 时钟拉高
    I2C_Delay()                // 延时
    dat <<= 1 → dat = 0000 0000（左移）
    SDA = 0（从机发送bit7=0）
    if(SDA)=false → dat不变 = 0000 0000

第2轮（i=1，接收bit6=1）：

    SCL = 0

第3轮（i=2，接收bit5=0）：
    SCL = 1
    I2C_Delay()
    dat <<= 1 → dat = 0000 0010（左移）
    SDA = 0
    if(SDA)=false → dat不变 = 0000 0010
    SCL = 0
    I2C_Delay()

第4轮（i=3，接收bit4=1）：
    SCL = 1
    I2C_Delay()
    dat <<= 1 → dat = 0000 0100
    SDA = 1
    if(SDA)=true → dat |= 0x01 → dat = 0000 0101
    SCL = 0
    I2C_Delay()

第5轮（i=4，接收bit3=0）：
    SCL = 1
    I2C_Delay()
    dat <<= 1 → dat = 0000 1010
    SDA = 0
    if(SDA)=false → dat不变 = 0000 1010
    SCL = 0
    I2C_Delay()

第6轮（i=5，接收bit2=1）：
    SCL = 1
    I2C_Delay()
    dat <<= 1 → dat = 0001 0100
    SDA = 1
    if(SDA)=true → dat |= 0x01 → dat = 0001 0101
    SCL = 0
    I2C_Delay()

第7轮（i=6，接收bit1=0）：
    SCL = 1
    I2C_Delay()
    dat <<= 1 → dat = 0010 1010
    SDA = 0
    if(SDA)=false → dat不变 = 0010 1010
    SCL = 0

    SCL = 1

    SDA = 1
    if(SDA)=true → dat |= 0x01 → dat = 0101 0101

返回：dat = 0x55

数据组装过程图解：
初始: 0000 0000
i=0:  左移=0000 0000, SDA=0 → dat=0000 0000
```

## 九、函数五：I2C_WaitAck - 等待应答

### 代码实现

```c
bit I2C_WaitAck(void)
{
    bit ack;
    SCL = 1;                     // SCL拉高
    I2C_Delay();                 // 延时
    ack = SDA;                   // 采样SDA状态
    SCL = 0;                     // SCL拉低
    I2C_Delay();                 // 延时
    return ack;                  // 返回应答状态(0=ACK, 1=NACK)
}
```

### 执行过程示例

```
调用：ack = I2C_WaitAck()

场景1：从机正常应答(ACK)
    SCL = 1                    // 时钟拉高
    I2C_Delay()                // 延时
    ack = SDA                  // 采样SDA=0（从机拉低）
    SCL = 0                    // 时钟拉低
    I2C_Delay()
    return 0                   // 返回ACK

场景2：从机无响应(NACK)
    SCL = 1                    // 时钟拉高
    I2C_Delay()                // 延时
    ack = SDA                  // 采样SDA=1（从机未拉低）
    SCL = 0                    // 时钟拉低
    I2C_Delay()
    return 1                   // 返回NACK

时序图解（ACK场景）：
SCL:  ────┐  ┌──────
         └──┘
SDA:  ────┐  ┌──────
         └──┘
         0=ACK（从机拉低SDA）
         
时序图解（NACK场景）：
SCL:  ────┐  ┌──────
         └──┘
SDA:  ────────────────
         1=NACK（保持高）
```

## 十、函数六：I2C_SendAck - 发送应答

### 代码实现

```c
void I2C_SendAck(bit ack)
{
    SCL = 0;                     // SCL拉低
    SDA = ack;                   // 0=ACK, 1=NACK
    I2C_Delay();                 // 延时
    SCL = 1;                     // SCL拉高，从机采样
    I2C_Delay();                 // 延时
    SCL = 0;                     // SCL拉低
    SDA = 1;                     // 释放SDA
    I2C_Delay();                 // 延时
}
```

### 执行过程示例

```
调用：I2C_SendAck(0)  // 发送ACK

步骤1：SCL = 0                    // 时钟拉低
步骤2：SDA = ack = 0             // 拉低SDA表示ACK
步骤3：I2C_Delay()               // 延时
步骤4：SCL = 1                   // 时钟拉高，从机看到ACK
步骤5：I2C_Delay()               // 延时
步骤6：SCL = 0                   // 时钟拉低
步骤7：SDA = 1                   // 释放SDA
步骤8：I2C_Delay()               // 延时

调用：I2C_SendAck(1)  // 发送NACK

步骤2：SDA = ack = 1             // 保持SDA高表示NACK
...

时序图解（ACK）：
SCL:  ────┐  ┌──────
         └──┘
SDA:  ────┐  ┌──────
         └──┘
         主机拉低SDA=ACK
         
时序图解（NACK）：
SCL:  ────┐  ┌──────
         └──┘
SDA:  ────────────────
         主机保持高=NACK
```

## 十三、六个基础函数对比总结

| 函数 | 功能 | 参数 | 返回值 | 关键操作 |
|------|------|------|--------|----------|
| **I2C_Start** | 启动通信 | 无 | 无 | SCL高时SDA下降沿 |
| **I2C_Stop** | 停止通信 | 无 | 无 | SCL高时SDA上升沿 |
| **I2C_SendByte** | 发送字节 | dat(8位) | 无 | MSB先发，SCL低改变SDA |
| **I2C_RecvByte** | 接收字节 | 无 | dat(8位) | 左移组装，SCL高采样 |
| **I2C_WaitAck** | 等待应答 | 无 | ack(0=ACK,1=NACK) | SCL高时采样SDA |
| **I2C_SendAck** | 发送应答 | ack(0/1) | 无 | SCL低时设置SDA |
