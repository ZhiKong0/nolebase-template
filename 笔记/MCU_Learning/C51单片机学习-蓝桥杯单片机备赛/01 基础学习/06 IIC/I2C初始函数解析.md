## 一、函数总览

赛题BSP（`I2C.c`）提供的**6个核心函数**：

| 函数名 | 功能 | 调用层级 |
|--------|------|----------|
| `I2C_Start` | I2C启动信号 | 基础时序 |
| `I2C_Stop` | I2C停止信号 | 基础时序 |
| `I2C_SendByte` | 发送一个字节 | 基础时序 |
| `I2C_RecvByte` | 接收一个字节 | 基础时序 |
| `I2C_WaitAck` | 等待从机应答 | 基础时序 |
| `I2C_SendAck` | 发送应答信号 | 基础时序 |

**上层应用函数**（项目特定扩展）：
| 函数名 | 功能 | 设备 |
|--------|------|------|
| `PCF8591_Read` | 读取AD值 | PCF8591 |
| `PCF8591_Write` | 输出DA值 | PCF8591 |
| `EEPROM_WriteByte` | 单字节写入 | AT24C02 |
| `EEPROM_ReadByte` | 单字节读取 | AT24C02 |

**调用关系**：
```
用户代码
   ↓ 调用
PCF8591_Read() / EEPROM_ReadByte()
   ↓ 内部调用
I2C_Start() → I2C_SendByte() → I2C_WaitAck() → 
I2C_RecvByte() → I2C_SendAck() → I2C_Stop()
   ↓ 底层操作
SCL(P20) / SDA(P21) 引脚时序
```

## 二、引脚定义与硬件连接

```c
sbit SCL = P2 ^ 0;   // I2C时钟线 (P2.0) → 所有I2C设备SCL
sbit SDA = P2 ^ 1;   // I2C数据线 (P2.1) → 所有I2C设备SDA
```

**CT107D开发板I2C总线连接图**：
```
STC15F2K60S2              I2C总线设备
    P2.0  ──────────┬──────► SCL (PCF8591)
                    ├──────► SCL (AT24C02)
    P2.1  ◄────┬────┴─────► SDA (PCF8591)
              │          ► SDA (AT24C02)
              │
         ┌────┴────┐
         │ 上拉电阻 │  (开发板已集成4.7K上拉)
         └────┬────┘
              │
            VCC
```

**多设备共享总线**：
- I2C是总线协议，所有设备并联连接
- 每个设备有唯一的7位地址
- 主机通过地址选择要通信的从机

## 三、I2C_Delay 延时函数解析

### 3.1 完整源码

```c
/**
 * @brief I2C时序延时函数
 * @details 产生约5us的延时，满足I2C标准时序要求
 */
#define DELAY_TIME 5

static void I2C_Delay(void)
{
    u8 i;
    for(i = 0; i < DELAY_TIME; i++)
    {
        _nop_();                     // 空操作，延时1个机器周期
    }
}
```

### 3.2 延时计算

| 晶振频率 | 机器周期 | DELAY_TIME=5 | 总延时 |
|----------|----------|--------------|--------|
| 12MHz | 1μs | 5 × 1μs | 约5μs |
| 11.0592MHz | 1.085μs | 5 × 1.085μs | 约5.4μs |

**为什么需要延时？**
- I2C标准规定SCL时钟频率不超过100KHz（标准模式）
- 周期10μs，高低电平各约5μs
- 单片机指令执行太快，需要适当延时满足时序要求
- 保证信号稳定，避免毛刺

## 四、I2C_Start 启动信号解析

### 4.1 完整源码

```c
/**
 * @brief I2C启动信号
 * @details 产生起始条件：SCL高电平时SDA从高变低
 */
void I2C_Start(void)
{
    SDA = 1;                         // 1. SDA先置高
    SCL = 1;                         // 2. SCL置高
    I2C_Delay();                     // 3. 延时
    SDA = 0;                         // 4. SCL高时SDA下降沿 = 启动信号
    I2C_Delay();                     // 5. 延时
    SCL = 0;                         // 6. SCL拉低，准备发送数据
}
```

### 4.2 时序详解

```
启动条件时序图解：

SDA:  ────┐              ┌────────────────
         └──────────────┘
              ↑ 下降沿
SCL:  ──────────────────┐
                       └──────

时间线分析：
T0: SDA=1, SCL=1        (空闲状态)
T1: I2C_Delay()         (延时约5μs)
T2: SDA=0               (在SCL=1时产生下降沿) ← 起始条件！
T3: I2C_Delay()         (延时)
T4: SCL=0               (拉低准备传输)
```

**关键要点**：
- 必须在SCL高电平时产生SDA下降沿
- 起始条件后，主机占用总线，可以开始传输数据
- SCL最后拉低，准备发送第一个数据位

## 五、I2C_Stop 停止信号解析

### 5.1 完整源码

```c
/**
 * @brief I2C停止信号
 * @details 产生停止条件：SCL高电平时SDA从低变高
 */
void I2C_Stop(void)
{
    SDA = 0;                         // 1. SDA先置低
    SCL = 1;                         // 2. SCL置高
    I2C_Delay();                     // 3. 延时
    SDA = 1;                         // 4. SCL高时SDA上升沿 = 停止信号
    I2C_Delay();                     // 5. 延时
}
```

### 5.2 时序详解

```
停止条件时序图解：

SDA:  ────┐              ┌────────────────
         └──────────────┘
                       ↑ 上升沿
SCL:  ──────────────────┐
                       └──────

时间线分析：
T0: SDA=0, SCL=?        (确保SDA为低)
T1: SCL=1               (拉高SCL)
T2: I2C_Delay()         (延时)
T3: SDA=1               (在SCL=1时产生上升沿) ← 停止条件！
T4: I2C_Delay()         (延时)
T5: SDA=1, SCL=1        (回到空闲状态)
```

**关键要点**：
- 必须在SCL高电平时产生SDA上升沿
- 停止条件后，总线释放，回到空闲状态
- 所有设备可以重新竞争总线

## 六、I2C_SendByte 发送字节解析

### 6.1 完整源码

```c
/**
 * @brief I2C发送一个字节
 * @param dat 要发送的8位数据
 * @details 高位先发(MSB)，SCL低电平改变SDA，SCL高电平稳定
 */
void I2C_SendByte(u8 dat)
{
    u8 i;
    for(i = 0; i < 8; i++)           // 循环8次，发送8位
    {
        SCL = 0;                     // SCL低电平，准备改变SDA
        if(dat & 0x80)               // 判断最高位(MSB)
            SDA = 1;
        else
            SDA = 0;
        I2C_Delay();                 // 延时
        SCL = 1;                     // SCL高电平，数据稳定，从机读取
        I2C_Delay();                 // 延时
        dat <<= 1;                   // 左移，准备发送下一位
    }
    SCL = 0;                         // SCL拉低，准备接收应答
}
```

### 6.2 位操作时序详解

```
发送 0x55 = 01010101 的时序图解：

i=0 (bit7=0):    i=1 (bit6=1):    i=2 (bit5=0):    i=3 (bit4=1):
SCL:  ┐ ┌─┐       ┐ ┌─┐       ┐ ┌─┐       ┐ ┌─┐
      └─┘ └─      └─┘ └─      └─┘ └─      └─┘ └─
SDA:  ──┐   ┌───┐   ┌───┐   ┌───┐   ┌───
          └───┘   └───┘   └───┘   └───
          0       1       0       1

i=4 (bit3=0):    i=5 (bit2=1):    i=6 (bit1=0):    i=7 (bit0=1):
SCL:  ┐ ┌─┐       ┐ ┌─┐       ┐ ┌─┐       ┐ ┌─┐
      └─┘ └─      └─┘ └─      └─┘ └─      └─┘ └─
SDA:  ──┐   ┌───┐   ┌───┐   ┌───┐   ┌───
          └───┘   └───┘   └───┘   └───
          0       1       0       1

发送顺序：0→1→0→1→0→1→0→1
对应位： bit7 bit6 bit5 bit4 bit3 bit2 bit1 bit0
```

### 6.3 关键代码解析

| 代码 | 作用 | 时序阶段 |
|------|------|----------|
| `SCL = 0` | 时钟拉低 | 准备阶段：允许改变SDA |
| `dat & 0x80` | 取最高位 | 判断要发送的位值 |
| `SDA = 1/0` | 设置数据线 | 放置数据到SDA |
| `SCL = 1` | 时钟拉高 | 数据稳定，从机采样 |
| `dat <<= 1` | 左移1位 | 准备发送下一位(MSB) |

**MSB FIRST说明**：
- I2C协议规定**高位先发（Most Significant Bit First）**
- `dat & 0x80` 取出最高位（bit7）
- `dat <<= 1` 左移后，原来的bit6变成新的bit7
- 循环8次后，从bit7到bit0依次发送完毕

## 七、I2C_RecvByte 接收字节解析

### 7.1 完整源码

```c
/**
 * @brief I2C接收一个字节
 * @retval 接收到的8位数据
 * @details 高位先收(MSB)，释放SDA后从机驱动，主机在SCL高电平时采样
 */
u8 I2C_RecvByte(void)
{
    u8 i, dat = 0;
    SDA = 1;                         // 释放SDA，由从机驱动
    for(i = 0; i < 8; i++)
    {
        SCL = 1;                     // SCL高电平，数据稳定
        I2C_Delay();                 // 延时
        dat <<= 1;                   // 左移，为下一位腾出位置
        if(SDA)                      // 采样SDA状态
            dat |= 0x01;             // SDA=1，最低位置1
        SCL = 0;                     // SCL低电平，从机准备下一位
        I2C_Delay();                 // 延时
    }
    return dat;
}
```

### 7.2 数据接收组装原理

```
数据组装过程（接收0x55 = 01010101）：

初始: dat = 0x00 = 0000 0000

i=0: SCL=1, SDA=0 (bit7)
     dat <<= 1  →  0000 0000
     SDA=0 → dat不变 → 0000 0000

i=1: SCL=1, SDA=1 (bit6)
     dat <<= 1  →  0000 0000
     SDA=1 → dat|=0x01 → 0000 0001

i=2: SCL=1, SDA=0 (bit5)
     dat <<= 1  →  0000 0010
     SDA=0 → dat不变 → 0000 0010

i=3: SCL=1, SDA=1 (bit4)
     dat <<= 1  →  0000 0100
     SDA=1 → dat|=0x01 → 0000 0101

i=4: SCL=1, SDA=0 (bit3)
     dat <<= 1  →  0000 1010
     SDA=0 → dat不变 → 0000 1010

i=5: SCL=1, SDA=1 (bit2)
     dat <<= 1  →  0001 0100
     SDA=1 → dat|=0x01 → 0001 0101

i=6: SCL=1, SDA=0 (bit1)
     dat <<= 1  →  0010 1010
     SDA=0 → dat不变 → 0010 1010

i=7: SCL=1, SDA=1 (bit0)
     dat <<= 1  →  0101 0100
     SDA=1 → dat|=0x01 → 0101 0101 ← 0x55！

接收图解：
初始:    0000 0000
bit7=0: 0000 0000
bit6=1: 0000 0001
bit5=0: 0000 0010
bit4=1: 0000 0101
bit3=0: 0000 1010
bit2=1: 0001 0101
bit1=0: 0010 1010
bit0=1: 0101 0101 ← 最终结果！
```

**为什么用 `dat <<= 1` 和 `dat |= 0x01`？**
- 因为是MSB先发，每个新bit要放到最高位
- `dat <<= 1` 把已接收的位往左推
- `dat |= 0x01` 在新腾出的最低位(bit0)填入新数据
- 经过8轮后，数据从bit7到bit0依次组装完成

## 八、I2C_WaitAck 等待应答解析

### 8.1 完整源码

```c
/**
 * @brief I2C等待应答信号
 * @retval 0: 收到应答(ACK)，1: 无应答(NACK)
 * @details 主机发送完8位后，释放SDA并检测从机是否拉低SDA
 */
bit I2C_WaitAck(void)
{
    bit ack;
    SCL = 1;                         // SCL拉高
    I2C_Delay();                     // 延时
    ack = SDA;                       // 采样SDA：0=ACK, 1=NACK
    SCL = 0;                         // SCL拉低
    I2C_Delay();                     // 延时
    return ack;
}
```

### 8.2 应答检测时序

```
ACK场景（从机正常响应）：

SCL:  ────┐  ┌──────
         └──┘
SDA:  ────┐  ┌──────
         └──┘
         从机拉低SDA=ACK

第9个时钟周期：
- 主机释放SDA（变为输入）
- 从机拉低SDA表示ACK
- 主机在SCL高电平时采样SDA=0
- 返回ack=0表示成功

NACK场景（从机无响应）：

SCL:  ────┐  ┌──────
         └──┘
SDA:  ─────────────────
         保持高=NACK

第9个时钟周期：
- 主机释放SDA
- 从机未拉低（保持高电平）
- 主机采样SDA=1
- 返回ack=1表示失败
```

## 九、I2C_SendAck 发送应答解析

### 9.1 完整源码

```c
/**
 * @brief I2C发送应答信号
 * @param ack 应答类型：0=ACK，1=NACK
 * @details 主机接收完8位数据后，通过SDA向从机发送应答
 */
void I2C_SendAck(bit ack)
{
    SCL = 0;                         // SCL拉低
    SDA = ack;                       // 0=ACK, 1=NACK
    I2C_Delay();                     // 延时
    SCL = 1;                         // SCL拉高，从机采样
    I2C_Delay();                     // 延时
    SCL = 0;                         // SCL拉低
    SDA = 1;                         // 释放SDA
    I2C_Delay();                     // 延时
}
```

### 9.2 发送ACK时序

```
发送ACK(ack=0)图解：

SCL:  ────┐  ┌──────
         └──┘
SDA:  ────┐  ┌──────
         └──┘
         主机拉低SDA=ACK
         
发送NACK(ack=1)图解：

SCL:  ────┐  ┌──────
         └──┘
SDA:  ─────────────────
         保持高=NACK

第9个时钟周期：
- SCL低时，主机设置SDA（0=ACK，1=NACK）
- SCL拉高，从机看到应答
- SCL拉低，主机释放SDA
```

## 十一、常见错误与调试

### 11.1 从机无应答(始终NACK)

**可能原因**：
- 设备地址错误（如把0x90写成0x91）
- 设备未连接或损坏
- SCL/SDA引脚接错
- 总线被其他设备占用
- 时序太快，设备来不及响应

**调试方法**：
```c
// 添加调试代码
bit ack = I2C_WaitAck();
if(ack) {
    // NACK，点亮错误指示灯
    LED_Error = 1;
    I2C_Stop();  // 停止通信
    return 0;    // 返回错误
}
```

### 11.2 读取数据错误

**可能原因**：
- PCF8591需要Dummy Read，第一次读取丢弃
- EEPROM读取地址设置错误（需要先写地址再读）
- 数据格式错误（BCD vs 二进制）

### 11.3 调试技巧

```c
// 在关键位置加LED指示
void I2C_SendByte_Debug(u8 dat)
{
    LED_On();                    // 开始发送指示
    
    for(i = 0; i < 8; i++) {
        // 正常发送代码...
        LED_Toggle();            // 每发送1位翻转LED
    }
    
    LED_Off();                   // 发送完成
}
```
