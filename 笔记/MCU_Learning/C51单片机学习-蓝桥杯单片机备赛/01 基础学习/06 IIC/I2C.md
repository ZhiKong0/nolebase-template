### 二、I2C总线概述

#### 2.1 I2C协议特点
- **核心特点**：两线串行总线（SCL时钟线 + SDA数据线）
- **通信方式**：同步、半双工、主从架构
- **设备数量**：总线上可挂接多个设备（理论127个，实际受电容限制）
- **寻址方式**：7位设备地址 + 1位读写标志

#### 2.2 硬件连接（CT107D开发板）

| I2C信号 | 单片机引脚 | 功能 | 连接设备 |
| ------- | ---------- | ---- | -------- |
| **SCL** | P20 | 串行时钟 | PCF8591、AT24C02 |
| **SDA** | P21 | 串行数据 | PCF8591、AT24C02 |

**I2C总线拓扑**（多设备共享）：
```
         ┌─────────┐
    SCL ─┤  PC8591 │──┐
    SDA ─┤ (0x90)  │  │
         └─────────┘  │
                      │
         ┌─────────┐  │
    SCL ─┤AT24C02  │──┼── SCL(P20)
    SDA ─┤ (0xA0)  │──┼── SDA(P21)
         └─────────┘  │
                      │
         ┌─────────┐  │
    SCL ─┤其他设备 │──┘
    SDA ─┤         │
         └─────────┘
```

**关键特性**：
- SDA为开漏输出，需外部上拉电阻（开发板已集成）
- 所有设备并联在总线上，通过地址区分
- 主机（单片机）发起通信，从机（外设）响应

### 三、I2C设备地址定义

| 设备 | 写地址 | 读地址 | 功能 |
|------|--------|--------|------|
| **PCF8591** | 0x90 | 0x91 | 4路AD输入 + 1路DA输出 |
| **AT24C02** | 0xA0 | 0xA1 | 256字节EEPROM存储 |

**地址结构**（7位地址 + 1位读写位）：
```
PCF8591:  0x90 = 1001 0000 → 设备地址=0x48(1001000), 读写位=0(写)
          0x91 = 1001 0001 → 设备地址=0x48(1001000), 读写位=1(读)

AT24C02: 0xA0 = 1010 0000 → 设备地址=0x50(1010000), 读写位=0(写)
         0xA1 = 1010 0001 → 设备地址=0x50(1010000), 读写位=1(读)
```

### 四、I2C时序详解

#### 4.1 起始条件（Start）
```
SCL:  高 ________
SDA:  高 ____\____ 低
         起始位
```
- **条件**：SCL高电平时，SDA从高变低
- **作用**：通知总线上所有设备，通信即将开始

#### 4.2 停止条件（Stop）
```
SCL:  高 ________
SDA:  低 ____/____ 高
         停止位
```
- **条件**：SCL高电平时，SDA从低变高
- **作用**：结束当前通信，释放总线

#### 4.3 数据位传输
```
SCL:  低_高_低_高_低_高_低
SDA:  D7 D6 D5 D4 D3 D2 D1 D0
```
- **SCL低电平**：主机改变SDA数据
- **SCL高电平**：数据稳定，从机采样
- **传输顺序**：高位先发（MSB First）

#### 4.4 应答信号（ACK/NACK）
```
SCL:  低_高_低
SDA:  ?____?
      ACK=0, NACK=1
```
- **ACK（应答）**：SDA=0，表示从机收到数据
- **NACK（非应答）**：SDA=1，表示从机未响应或通信结束

### 五、PCF8591 AD/DA转换器

#### 5.1 芯片功能
- **4路模拟输入**（AIN0-AIN3）：
  - 通道0：扩展接口/未用
  - 通道1：光敏电阻（测量光照强度）
  - 通道2：LM324/差分输入
  - 通道3：电位器Rb2（可调电压输入）
- **1路模拟输出**（AOUT）：DA转换输出
- **分辨率**：8位（0-255）
- **转换时间**：约10μs

#### 5.2 控制字节格式
```
┌────┬────┬────┬────┬────┬────┬────┬────┐
│bit7│bit6│bit5│bit4│bit3│bit2│bit1│bit0│
├────┼────┼────┼────┼────┼────┼────┼────┤
│ 0  │DAEN│ 0  │AIN│CH1 │CH0 │ 0  │ 0  │
└────┴────┴────┴────┴────┴────┴────┴────┘
```

| 位 | 名称 | 功能 |
|----|------|------|
| bit7 | - | 固定为0 |
| bit6 | DAEN | 1=使能DA输出，0=禁止DA输出 |
| bit5 | - | 固定为0 |
| bit4 | AIN | 1=差分输入，0=单端输入 |
| bit3-bit2 | CH1-CH0 | 通道选择：00=CH0, 01=CH1, 10=CH2, 11=CH3 |
| bit1-bit0 | - | 固定为00 |

**常用控制字**：
- `0x00`：单端输入，通道0，AD转换
- `0x01`：单端输入，通道1（光敏电阻）
- `0x03`：单端输入，通道3（电位器）
- `0x40`：使能DA输出

#### 5.3 AD读取流程
```
启动 -> 发送地址(0x90写) -> ACK -> 发送控制字(通道) -> ACK
启动 -> 发送地址(0x91读) -> ACK -> 读取第1字节(丢弃) -> ACK
-> 读取第2字节(有效数据) -> NACK -> 停止
```
**注意**：PCF8591需要Dummy Read，第一次读取的值是上次转换结果，第二次才是当前值。

#### 5.4 DA输出流程
```
启动 -> 发送地址(0x90写) -> ACK -> 发送控制字(0x40) -> ACK
-> 发送DA值(0-255) -> ACK -> 停止
```

### 六、AT24C02 EEPROM存储器

#### 6.1 芯片参数
- **容量**：256字节（2Kbit = 256×8bit）
- **页大小**：8字节（一次最多连续写8字节）
- **写周期**：约5ms（写入后需延时）
- **擦写寿命**：约10万次

#### 6.2 单字节写入流程
```
启动 -> 发送地址(0xA0写) -> ACK -> 发送存储地址(0-255) -> ACK
-> 发送数据字节 -> ACK -> 停止 -> 延时5ms
```

#### 6.3 单字节读取流程
```
启动 -> 发送地址(0xA0写) -> ACK -> 发送存储地址(0-255) -> ACK
启动 -> 发送地址(0xA1读) -> ACK -> 读取数据 -> NACK -> 停止
```

#### 6.4 页写入注意事项
- 一次最多写入8字节（一页）
- 写入地址不能跨页（地址0x07和0x08属于不同页）
- 跨页写入需要分多次调用

### 七、代码实现步骤

#### 7.1 官方驱动文件引入
蓝桥杯官方资源包提供I2C底层时序函数：
```c
void I2C_Start(void);           // 启动信号
void I2C_Stop(void);            // 停止信号
void I2C_SendByte(u8 dat);      // 发送字节
u8   I2C_RecvByte(void);        // 接收字节
bit  I2C_WaitAck(void);         // 等待应答
void I2C_SendAck(bit ack);      // 发送应答
```

#### 7.2 引脚定义
```c
sbit SCL = P2^0;   // I2C时钟线
sbit SDA = P2^1;   // I2C数据线
```

#### 7.3 PCF8591读取函数
```c
/**
 * @brief PCF8591读取指定通道的AD值
 * @param channel 通道号0-3
 * @retval 8位AD转换结果0-255
 */
u8 PCF8591_Read(u8 channel)
{
    u8 dat;
    
    // 第一次通信：发送控制字，选择通道
    I2C_Start();
    I2C_SendByte(0x90);              // PCF8591写地址
    I2C_WaitAck();
    I2C_SendByte(0x00 | (channel & 0x03));  // 选择通道，AD模式
    I2C_WaitAck();
    
    // 第二次通信：读取AD值
    I2C_Start();
    I2C_SendByte(0x91);              // PCF8591读地址
    I2C_WaitAck();
    (void)I2C_RecvByte();            // 第一次读取丢弃（Dummy Read）
    I2C_SendAck(0);                  // 发送ACK，继续读取
    dat = I2C_RecvByte();            // 第二次读取为有效数据
    I2C_SendAck(1);                  // 发送NACK，结束读取
    I2C_Stop();
    
    return dat;
}
```

#### 7.4 PCF8591 DA输出函数
```c
/**
 * @brief PCF8591输出DA值
 * @param dat DA输出值0-255
 */
void PCF8591_Write(u8 dat)
{
    I2C_Start();
    I2C_SendByte(0x90);              // PCF8591写地址
    I2C_WaitAck();
    I2C_SendByte(0x40);              // 使能DA输出
    I2C_WaitAck();
    I2C_SendByte(dat);               // DA输出值
    I2C_WaitAck();
    I2C_Stop();
}
```

#### 7.5 EEPROM单字节读写
```c
/**
 * @brief EEPROM写入一个字节
 * @param addr 存储地址0-255
 * @param dat 要写入的数据
 */
void EEPROM_WriteByte(u8 addr, u8 dat)
{
    I2C_Start();
    I2C_SendByte(0xA0);              // AT24C02写地址
    I2C_WaitAck();
    I2C_SendByte(addr);              // 存储地址
    I2C_WaitAck();
    I2C_SendByte(dat);               // 数据
    I2C_WaitAck();
    I2C_Stop();
    
    // 写周期延时（>5ms）
    Delay_ms(5);
}

/**
 * @brief EEPROM读取一个字节
 * @param addr 存储地址0-255
 * @retval 读取到的数据
 */
u8 EEPROM_ReadByte(u8 addr)
{
    u8 dat;
    
    // 先发送存储地址
    I2C_Start();
    I2C_SendByte(0xA0);              // AT24C02写地址
    I2C_WaitAck();
    I2C_SendByte(addr);              // 存储地址
    I2C_WaitAck();
    
    // 再读取数据
    I2C_Start();
    I2C_SendByte(0xA1);              // AT24C02读地址
    I2C_WaitAck();
    dat = I2C_RecvByte();            // 读取数据
    I2C_SendAck(1);                  // 发送NACK
    I2C_Stop();
    
    return dat;
}
```

### 八、关键注意事项

1. **I2C时序延时**：
   - SCL时钟不能太快，需适当延时（建议5-10μs）
   - 使用`_nop_()`或延时函数实现

2. **Dummy Read机制**：
   - PCF8591需要两次读取，第一次丢弃
   - 原因是芯片内部ADC转换需要时间

3. **EEPROM写周期**：
   - 写入后必须延时5ms以上
   - 连续写入同一地址会损耗寿命（10万次）

4. **页写入边界**：
   - AT24C02一页8字节
   - 地址0x07后不能直接写0x08（跨页）

5. **ACK检测**：
   - 每次发送后必须检测ACK
   - ACK=0表示正常，ACK=1表示设备无响应

6. **总线释放**：
   - 通信结束必须发送停止条件
   - 异常情况下需手动释放SDA
