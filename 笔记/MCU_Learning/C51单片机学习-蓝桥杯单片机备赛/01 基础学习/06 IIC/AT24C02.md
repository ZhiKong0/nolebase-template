### 二、AT24C02芯片概述

#### 2.1 芯片功能
- **核心功能**：电可擦除可编程只读存储器（EEPROM）
- **存储容量**：256字节（2Kbit = 256×8bit）
- **接口类型**：I2C两线串行总线
- **关键特性**：掉电后数据不丢失，可长期保存

#### 2.2 硬件连接（CT107D开发板）

| AT24C02引脚 | 连接 | 功能 |
|-------------|------|------|
| **VCC** | 3.3V/5V | 电源正 |
| **GND** | GND | 电源地 |
| **SCL** | P20 | I2C时钟线 |
| **SDA** | P21 | I2C数据线 |
| **WP** | GND | 写保护（低电平=可写） |
| **A0-A2** | GND | 地址引脚（接地=地址0xA0） |

**I2C总线连接图**：
```
STC15F2K60S2          AT24C02芯片
    P2.0  ──────────► SCL (时钟输入)
    P2.1  ◄────————─► SDA (双向数据线)
         
         ┌─────────┐
         │ 上拉电阻 │  (4.7K，开发板已集成)
         └────┬────┘
              │
            VCC
```

### 三、I2C设备地址

| 设备 | 写地址 | 读地址 | 说明 |
|------|--------|--------|------|
| **AT24C02** | 0xA0 | 0xA1 | 7位地址0x50 + 读写位 |

**地址结构解析**：
```
0xA0 = 1010 0000
       │││││││└─ 读写位：0=写
       │││││└┴┴── 器件地址：A2 A1 A0（接地=000）
       └┴┴┴────── 固定前缀：1010（EEPROM标识）

0xA1 = 1010 0001
                         └─ 读写位：1=读
```

**注意**：开发板上A0-A2引脚都接地，所以7位器件地址固定为0x50。

### 四、存储器结构

#### 4.1 容量与分页
- **总容量**：256字节
- **页大小**：8字节/页
- **总页数**：32页（256÷8=32）

**地址分布**：
```
页0：  地址 0x00-0x07
页1：  地址 0x08-0x0F
页2：  地址 0x10-0x17
...
页31：地址 0xF8-0xFF
```

#### 4.2 页写入边界
**重要**：一次页写入不能超过8字节，且不能跨页。
```
错误示例（跨页）：
从0x06开始写4字节 → 会覆盖0x06,0x07,0x08,0x09
但0x06,0x07属于页0，0x08,0x09属于页1
这会导致数据错误！

正确做法：
从0x00开始写8字节 → 0x00-0x07（都在页0）✓
从0x08开始写8字节 → 0x08-0x0F（都在页1）✓
```

### 五、写周期与寿命

#### 5.1 写周期延时
- **写入后延时**：必须延时5ms以上
- **原因**：EEPROM内部擦写需要时间
- **后果**：不延时直接读取可能得到旧数据

#### 5.2 擦写寿命
- **寿命**：约10万次擦写
- **注意**：频繁写入同一地址会损坏存储单元
- **建议**：使用磨损均衡算法（循环写入不同地址）

### 六、I2C通信时序

#### 6.1 单字节写入流程
```
启动 -> 发送设备地址(0xA0) -> ACK -> 发送存储地址(0-255) -> ACK
-> 发送数据字节 -> ACK -> 停止 -> 延时5ms
```

#### 6.2 单字节读取流程
```
启动 -> 发送设备地址(0xA0) -> ACK -> 发送存储地址(0-255) -> ACK
启动 -> 发送设备地址(0xA1) -> ACK -> 读取数据 -> NACK -> 停止
```

#### 6.3 页写入流程
```
启动 -> 发送设备地址(0xA0) -> ACK -> 发送起始地址 -> ACK
-> 发送数据1 -> ACK -> 发送数据2 -> ACK ...
-> 发送数据N -> ACK -> 停止 -> 延时5ms
（N≤8，且不能跨页）
```

### 七、代码实现

#### 7.1 单字节写入函数
```c
/**
 * @brief EEPROM写入一个字节
 * @param addr 存储地址，范围0-255
 * @param dat 要写入的数据
 * @note 写入后自动延时等待写周期完成（约5ms）
 */
void EEPROM_WriteByte(u8 addr, u8 dat)
{
    I2C_Start();
    I2C_SendByte(0xA0);         // AT24C02写地址
    I2C_WaitAck();
    I2C_SendByte(addr);         // 存储地址
    I2C_WaitAck();
    I2C_SendByte(dat);           // 数据
    I2C_WaitAck();
    I2C_Stop();
    
    // 写周期延时（>5ms）
    Delay_ms(5);
}
```

#### 7.2 单字节读取函数
```c
/**
 * @brief EEPROM读取一个字节
 * @param addr 存储地址，范围0-255
 * @retval 读取到的数据
 */
u8 EEPROM_ReadByte(u8 addr)
{
    u8 dat;
    
    // 先发送存储地址
    I2C_Start();
    I2C_SendByte(0xA0);         // 写地址
    I2C_WaitAck();
    I2C_SendByte(addr);         // 存储地址
    I2C_WaitAck();
    
    // 再读取数据
    I2C_Start();
    I2C_SendByte(0xA1);         // 读地址
    I2C_WaitAck();
    dat = I2C_RecvByte();       // 读取数据
    I2C_SendAck(1);             // 发送NACK
    I2C_Stop();
    
    return dat;
}
```

#### 7.3 页写入函数
```c
/**
 * @brief EEPROM连续写入多个字节（页写入）
 * @param addr 起始地址（建议8字节对齐）
 * @param buf 数据缓冲区指针
 * @param len 写入长度（不超过8字节）
 * @note 连续写入时不能超过页边界（8字节一页）
 */
void EEPROM_WriteBuf(u8 addr, u8 *buf, u8 len)
{
    // 检查是否跨页
    if((addr % 8) + len > 8)
    {
        // 跨页了！需要分两次写入
        // 这里简化处理，实际应该实现分块写入
    }
    
    I2C_Start();
    I2C_SendByte(0xA0);
    I2C_WaitAck();
    I2C_SendByte(addr);
    I2C_WaitAck();
    
    while(len--)
    {
        I2C_SendByte(*buf++);
        I2C_WaitAck();
    }
    
    I2C_Stop();
    
    // 写周期延时
    Delay_ms(5);
}
```

#### 7.4 连续读取函数
```c
/**
 * @brief EEPROM连续读取多个字节
 * @param addr 起始地址，范围0-255
 * @param buf 数据缓冲区指针
 * @param len 读取长度（不超过256字节）
 * @note 可以跨页连续读取
 */
void EEPROM_ReadBuf(u8 addr, u8 *buf, u8 len)
{
    // 发送存储地址
    I2C_Start();
    I2C_SendByte(0xA0);
    I2C_WaitAck();
    I2C_SendByte(addr);
    I2C_WaitAck();
    
    // 连续读取
    I2C_Start();
    I2C_SendByte(0xA1);
    I2C_WaitAck();
    
    while(len--)
    {
        *buf++ = I2C_RecvByte();
        I2C_SendAck(len == 0);  // 最后一个字节发送NACK(1)，其余ACK(0)
    }
    
    I2C_Stop();
}
```

### 八、应用实例

#### 8.1 存储按键次数（掉电保持）
```c
u8 key_count;

void main(void)
{
    System_Init();
    
    // 上电时读取之前的次数
    key_count = EEPROM_ReadByte(0x00);
    
    while(1)
    {
        if(Key_Scan() == KEY_PRESS)
        {
            key_count++;
            EEPROM_WriteByte(0x00, key_count);  // 保存
        }
        
        Nixie_DisplayNumber(key_count);
    }
}
```

#### 8.2 存储系统参数
```c
// 参数结构
#define PARAM_ADDR  0x10

typedef struct {
    u8 brightness;      // 亮度
    u8 contrast;        // 对比度
    u8 volume;          // 音量
    u8 reserved;        // 保留
} SystemParams;

void SaveParams(SystemParams *params)
{
    EEPROM_WriteBuf(PARAM_ADDR, (u8*)params, sizeof(SystemParams));
}

void LoadParams(SystemParams *params)
{
    EEPROM_ReadBuf(PARAM_ADDR, (u8*)params, sizeof(SystemParams));
}
```

#### 8.3 数据记录器（循环写入）
```c
#define RECORD_ADDR 0x20
#define RECORD_SIZE 8
#define MAX_RECORDS 10

u8 current_record = 0;

void RecordData(u8 *data)
{
    u8 addr = RECORD_ADDR + (current_record * RECORD_SIZE);
    EEPROM_WriteBuf(addr, data, RECORD_SIZE);
    
    current_record++;
    if(current_record >= MAX_RECORDS)
        current_record = 0;  // 循环覆盖
}
```

### 九、关键注意事项

1. **写周期延时**：
   - 写入后必须延时5ms以上
   - 可用`_nop_()`或`Delay_ms()`实现
   - 不延时会导致数据写入失败

2. **页写入边界**：
   - AT24C02一页8字节
   - 地址0x07后不能直接写0x08（跨页）
   - 跨页写入需要分多次调用

3. **擦写寿命**：
   - 约10万次擦写
   - 避免频繁写入同一地址
   - 重要数据建议使用磨损均衡

4. **读取不消耗寿命**：
   - 可以无限次读取
   - 读取不需要延时

5. **总线应答**：
   - 每次发送后必须检测ACK
   - ACK=0表示正常，ACK=1表示设备无响应

6. **写保护**：
   - 开发板上WP引脚接地（可写）
   - 如果WP接高电平，芯片只读

### 十、常见问题

**Q1: 写入后读取不正确？**
- 检查是否延时5ms以上
- 检查写入地址和读取地址是否一致
- 检查是否跨页写入

**Q2: 设备无应答？**
- 检查设备地址（0xA0/0xA1）
- 检查WP引脚是否接地
- 检查SCL/SDA连接是否正确

**Q3: 如何写入超过8字节的数据？**
- 分多次页写入
- 每次检查页边界
- 或使用连续单字节写入

**Q4: 数据保存多久？**
- EEPROM数据掉电后长期保持（10年以上）
- 适合保存配置参数、计数值等
