# DS1302 赛题BSP初始函数解析

> 蓝桥杯CT107D开发板 - 赛题提供的底层驱动函数源码级解析

---

## 一、函数总览

赛题BSP（`ds1302.c`）提供的**3个核心函数**：

| 函数名 | 功能 | 调用层级 |
|--------|------|----------|
| `Write_Ds1302` | 底层：向DS1302写入1字节数据（位操作） | 最底层 |
| `Write_Ds1302_Byte` | 中层：向指定地址写入1字节数据 | 调用底层 |
| `Read_Ds1302_Byte` | 中层：从指定地址读取1字节数据 | 调用底层 |

**调用关系**：
```
用户代码
   ↓ 调用
Write_Ds1302_Byte(address, dat)
   ↓ 内部调用
Write_Ds1302(address)  →  Write_Ds1302(dat)
   ↓ 底层位操作
SCK/SDA/RST 引脚时序
```

## 二、引脚定义与硬件连接

```c
sbit SCK = P1 ^ 7;   // 串行时钟线 (P1.7) → DS1302.SCLK
sbit SDA = P2 ^ 3;   // 串行数据线 (P2.3) → DS1302.IO
sbit RST = P1 ^ 3;   // 复位/使能线 (P1.3) → DS1302.CE
```

**CT107D开发板连接图**：
```
STC15F2K60S2          DS1302芯片
    P1.7  ──────────► SCK (时钟输入)
    P2.3  ◄────——───► IO  (双向数据线)
    P1.3  ──────────► CE  (片选使能，高有效)
```

## 三、Write_Ds1302 底层函数解析

### 3.1 完整源码

```c
/**
 * @brief DS1302底层位写入函数
 * @param temp 要写入的8位数据
 * 
 * 通信协议：SPI-like（类SPI三线通信）
 * - 时钟极性：CPOL=0（空闲时SCK为低）
 * - 时钟相位：CPHA=0（上升沿采样）
 * - 数据顺序：LSB FIRST（最低位先发）
 * - 数据宽度：8位
 */
void Write_Ds1302(unsigned char temp)
{
	unsigned char i;
	for (i = 0; i < 8; i++)		    // 循环8次，每次写1位
	{
		SCK = 0;				    // 1. SCK拉低，准备数据（下降沿后DS1302采样完成）
		SDA = temp & 0x01;		    // 2. 取出temp最低位(bit0)，放到SDA
		temp >>= 1;				    // 3. temp右移1位，准备下一位数据
		SCK = 1;				    // 4. SCK拉高，DS1302在上升沿采样SDA
	}
}
```

### 3.2 位操作时序详解

**写入1字节（以 0x55 = 0b01010101 为例）**：

```
时序图（LSB先发，bit0 → bit7）：

      ___     ___     ___     ___     ___     ___     ___     ___
SCK _/   \___/   \___/   \___/   \___/   \___/   \___/   \___/   \__
      |  1  |  0  |  1  |  0  |  1  |  0  |  1  |  0  |
SDA __1_____0_____1_____0_____1_____0_____1_____0________________
      bit0  bit1  bit2  bit3  bit4  bit5  bit6  bit7
      (LSB)                           (MSB)

数据发送顺序：1→0→1→0→1→0→1→0
对应二进制：  bit0 bit1 bit2 bit3 bit4 bit5 bit6 bit7
              1    0    1    0    1    0    1    0    = 0x55 (LSB FIRST)
```

### 3.3 关键代码解析

| 代码 | 作用 | 时序阶段 |
|------|------|----------|
| `SCK = 0` | 时钟拉低 | 准备阶段：DS1302等待数据 |
| `SDA = temp & 0x01` | 取最低位输出 | 放置数据到数据线 |
| `temp >>= 1` | 右移1位 | 准备下一位数据 |
| `SCK = 1` | 时钟拉高 | 上升沿：DS1302采样锁存 |

**LSB FIRST说明**：
- DS1302协议规定**最低有效位(LSB)先发**
- `0x55` 发送顺序：`bit0=1` → `bit1=0` → `bit2=1` → ... → `bit7=0`
- 右移操作 `>>= 1` 每次把高位移到低位，正好配合LSB FIRST顺序

## 四、Write_Ds1302_Byte 写地址+数据函数解析

### 4.1 完整源码

```c
/**
 * @brief DS1302单寄存器写入函数（地址+数据）
 * @param address 寄存器地址（如0x80表示秒寄存器写地址）
 * @param dat 要写入的数据（BCD编码）
 * 
 * 完整通信流程：
 * 1. RST=0, SCK=0：初始状态，总线复位
 * 2. RST=1：启动传输，使能DS1302（CE高有效）
 * 3. Write_Ds1302(address)：发送命令字节（地址+读/写标志）
 * 4. Write_Ds1302(dat)：发送数据字节
 * 5. RST=0：结束传输
 * 
 * 注意：DS1302采用"命令+数据"的双字节传输模式
 */
void Write_Ds1302_Byte(unsigned char address, unsigned char dat)
{
	//========== 阶段1：总线初始化 ==========
	RST = 0;
	_nop_();					// _nop_() = 空操作，延时1个机器周期
	SCK = 0;
	_nop_();
	
	//========== 阶段2：启动传输 ==========
	RST = 1;
	_nop_();					// RST拉高，使能DS1302（CE有效）
	
	//========== 阶段3：发送命令字节（地址） ==========
	Write_Ds1302(address);		// 告诉DS1302要操作哪个寄存器
	
	//========== 阶段4：发送数据字节 ==========
	Write_Ds1302(dat);			// 写入实际数据
	
	//========== 阶段5：结束传输 ==========
	RST = 0;
}
```

### 4.2 通信时序图

```
完整写入时序（以向秒寄存器0x80写入0x55为例）：

RST ───────┐                                      ┌────────
           │                                      │
           └──────────────────────────────────────┘
           ↑                                      ↑
         启动传输                                结束传输

SCK ────┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐  ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
        └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘  └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─
        │地址字节││地址字节││地址字节│...│地址字节││数据字节│...│数据字节│
        │ bit0  ││ bit1  ││ bit2   │   │ bit7  ││ bit0  │   │ bit7  │
        └───────┘└───────┘└────────┘   └───────┘└───────┘   └───────┘
        
SDA ────┐    ┌──┐    ┌──┐          ┌───┐    ┌──┐    ┌──┐
        └────┘  └────┘  └──────────┘   └────┘  └────┘  └──────────
        │1000│  │0000│  │ ...        │0101│  │0101│  │
        └────┘  └────┘  └──────────┘   └────┘  └────┘  └──────────
        地址=0x80                        数据=0x55
        (LSB FIRST: 0000 0001 → 实际发送 1000 0000)
        
时间线：
T0: RST=0, SCK=0    (初始状态)
T1: RST=1           (启动传输)
T2-T9: 发送地址字节 0x80
T10-T17: 发送数据字节 0x55
T18: RST=0          (结束传输)
```

### 4.3 _nop_() 的作用

```c
_nop_();  // 汇编空操作指令，延时1个机器周期
```

| 晶振频率 | 机器周期 | _nop_() 延时 |
|----------|----------|--------------|
| 12MHz | 1μs | 约1μs |
| 11.0592MHz | 1.085μs | 约1.085μs |

**为什么需要 _nop_()？**
- DS1302有严格的时序要求（最小脉冲宽度、建立时间、保持时间）
- 单片机执行速度太快，需要适当延时满足DS1302的时序要求
- 保证信号稳定，避免毛刺导致通信失败

## 五、Read_Ds1302_Byte 读数据函数解析

### 5.1 完整源码

```c
/**
 * @brief DS1302单寄存器读取函数
 * @param address 寄存器地址（读地址，如0x81表示秒寄存器读地址）
 * @return 读取到的8位数据（BCD编码）
 * 
 * 读取时序（注意与写入的区别）：
 * 1. 先发命令字节（地址+读标志），告诉DS1302要读哪个寄存器
 * 2. DS1302在SCK下降沿后将数据放到SDA线上
 * 3. 单片机在SCK上升沿采样SDA数据
 * 4. 读取顺序同样是LSB（最低位）先发
 * 
 * 数据接收原理：
 * - temp初始为0x00
 * - 每次循环temp右移1位（temp >>= 1）
 * - 如果SDA为高，将temp最高位(MSB)置1（temp |= 0x80）
 * - 8次循环后，temp获得完整的1字节数据
 */
unsigned char Read_Ds1302_Byte(unsigned char address)
{
	unsigned char i, temp = 0x00;
	
	//========== 阶段1：总线初始化 ==========
	RST = 0;
	_nop_();
	SCK = 0;
	_nop_();
	
	//========== 阶段2：启动传输 ==========
	RST = 1;
	_nop_();
	
	//========== 阶段3：发送读命令（地址） ==========
	Write_Ds1302(address);		// 告诉DS1302要读取哪个寄存器
	
	//========== 阶段4：读取数据字节（关键差异） ==========
	for (i = 0; i < 8; i++)		// 循环8次，每次读1位
	{
		SCK = 0;				// SCK下降沿，DS1302将数据放到SDA
		temp >>= 1;				// temp右移，腾出最高位(MSB)给新数据
		if (SDA)
			temp |= 0x80;		// 如果SDA为高，设置temp的最高位(bit7)
		SCK = 1;				// SCK上升沿，采样完成
	}
	
	//========== 阶段5：结束传输与总线复位 ==========
	RST = 0;
	_nop_();
	
	// 总线复位序列（释放总线，避免影响下次通信）
	SCK = 0;
	_nop_();
	SCK = 1;
	_nop_();
	SDA = 0;
	_nop_();
	SDA = 1;
	_nop_();
	
	return temp;				// 返回读取的数据
}
```

### 5.2 读取时序详解

```
读取时序图（从秒寄存器0x81读取数据）：

阶段1-3：同写入（RST拉高 → 发送地址0x81）

阶段4：读取数据（关键差异在此）

SCK ──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──
      └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘
      ↓  ↑  ↓  ↑  ↓  ↑  ↓  ↑  ↓  ↑  ↓  ↑  ↓  ↑  ↓  ↑  ↓  ↑
      下降沿  上升沿  下降沿  上升沿...（8次循环）
      
      ↓：DS1302输出数据到SDA
      ↑：单片机采样SDA数据
      
SDA ──╲  ╱╲  ╱╲  ╱╲  ╱╲  ╱╲  ╱╲  ╱╲  ╱╲
       ╲╱  ╲╱  ╲╱  ╲╱  ╲╱  ╲╱  ╲╱  ╲╱  ╲╱
       bit0 bit1 bit2 bit3 bit4 bit5 bit6 bit7
       (LSB)                    (MSB)

数据接收过程（以接收0x55 = 0b01010101为例）：
┌───────────┬───────────┬───────────────┬────────────────┐
│ 循环次数  │ SDA状态   │ temp右移后    │ temp|=0x80后   │
├───────────┼───────────┼───────────────┼────────────────┤
│ i=0       │ 1 (bit0)  │ 0x00 >> 1     │ 0x00|0x80=0x80 │
│           │           │ = 0x00        │ (bit7=1)       │
├───────────┼───────────┼───────────────┼────────────────┤
│ i=1       │ 0 (bit1)  │ 0x80 >> 1     │ 0x40|0x00=0x40 │
│           │           │ = 0x40        │ (bit7=0)       │
├───────────┼───────────┼───────────────┼────────────────┤
│ i=2       │ 1 (bit2)  │ 0x40 >> 1     │ 0x20|0x80=0xA0 │
│           │           │ = 0x20        │ (bit7=1)       │
├───────────┼───────────┼───────────────┼────────────────┤
│ ...       │ ...       │ ...           │ ...            │
├───────────┼───────────┼───────────────┼────────────────┤
│ i=7       │ 0 (bit7)  │ 0xAA >> 1     │ 0x55|0x00=0x55 │
│           │           │ = 0x55        │ (最终结果)     │
└───────────┴───────────┴───────────────┴────────────────┘
```

### 5.3 读取数据组装原理

**为什么用 `temp >>= 1` 和 `temp |= 0x80`？**

```
数据组装过程（接收0x55 = 0b01010101，LSB先发）：

初始: temp = 0x00 = 0b00000000

i=0: SDA=1 (bit0)
     temp >>= 1  →  0b00000000 (右移)
     temp |= 0x80 → 0b10000000 = 0x80 (设置MSB)

i=1: SDA=0 (bit1)
     temp >>= 1  →  0b01000000 (右移)
     temp |= 0x00 → 0b01000000 = 0x40 (MSB保持0)

i=2: SDA=1 (bit2)
     temp >>= 1  →  0b00100000 (右移)
     temp |= 0x80 → 0b10100000 = 0xA0 (设置MSB)

...（继续8次）

i=7: SDA=0 (bit7)
     temp >>= 1  →  0b00101010 (右移)
     temp |= 0x00 → 0b00101010 = 0x55 (最终结果！)
     
奇迹：8个bit0~bit7按LSB FIRST顺序装入temp，
     最终temp = 0b01010101 = 0x55 ✓
```

## 六、三个函数对比总结

| 特性 | Write_Ds1302 | Write_Ds1302_Byte | Read_Ds1302_Byte |
|------|--------------|-------------------|------------------|
| **功能** | 底层位操作 | 写地址+数据 | 读地址+数据 |
| **参数** | 1字节数据 | 地址+数据 | 地址 |
| **返回值** | 无 | 无 | 读取的数据 |
| **循环次数** | 8次（位循环） | 调用2次Write_Ds1302 | 1次Write+8次位操作 |
| **RST控制** | 无 | 有（启动/结束） | 有（启动/结束） |
| **时序** | 纯时钟+数据 | 命令+数据 | 命令+读数据 |
| **用户调用** | ❌ 一般不直接调用 | ✅ 写时间时调用 | ✅ 读时间时调用 |

## 七、常见错误与调试

### 7.1 读取数据全为0

**可能原因**：
- RST引脚未正确拉高（检查P1.3连接）
- SCK时钟太快（适当增加_nop_()延时）
- 地址错误（确认读地址=写地址+1）

### 7.2 写入后读取不正确

**可能原因**：
- 未关闭写保护（写入前必须`Write_Ds1302_Byte(0x8E, 0x00)`）
- BCD编码错误（0x09不是0x9）
- 数据未真正写入（检查RST时序）

### 7.3 调试技巧

```c
// 在关键位置加LED指示，排查通信问题
void Write_Ds1302_Byte_Debug(unsigned char address, unsigned char dat)
{
    LED_On();					// 开始通信指示灯
    
    RST = 0;
    _nop_();
    SCK = 0;
    _nop_();
    RST = 1;
    
    LED_Off();					// RST拉高完成
    
    Write_Ds1302(address);
    Write_Ds1302(dat);
    
    RST = 0;
}
```
