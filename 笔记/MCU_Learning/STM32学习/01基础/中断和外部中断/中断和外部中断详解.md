# STM32 中断系统与外部中断 (EXTI) 详解

> 基于STM32F1系列，深入理解中断机制、NVIC控制器与外部中断应用

## 一、课程导入与实验现象

### 1.1 实验演示

本节课通过两个实验展示外部中断的应用：

#### 实验一：红外对射传感器计次
- **硬件连接**: 红外对管传感器接入STM32引脚
- **现象**: OLED显示数字，挡光片遮挡红外对管时数字+1
- **原理**: 遮挡时输出电平从低到高产生**上升沿**，触发外部中断，执行计数程序

#### 实验二：旋转编码器计次
- **硬件连接**: 旋转编码器接入STM32两个引脚
- **现象**: 旋转编码器时，OLED显示的数字跟随变化
- **原理**: 旋转产生脉冲波形，通过外部中断捕获脉冲进行计数

## 二、中断基本概念

### 2.1 什么是中断？

**定义**: 在主程序运行过程中，出现特定的**中断触发条件**（中断源），使得CPU暂停当前程序，转而去执行中断处理程序，处理完成后返回原来被暂停的位置继续运行。

**通俗理解**: 就像定闹钟，定好后可以放心睡觉（执行主程序），时间到了闹钟响（中断触发），起床处理事情（执行中断程序），处理完回去继续睡（返回主程序）。

### 2.2 为什么需要中断？

| 场景 | 无中断系统 | 有中断系统 |
|------|-----------|-----------|
| 外部信号 | 主程序必须不断查询，无法干其他事 | 主程序执行其他任务，信号来时自动处理 |
| 串口接收 | 不立即读取会被下一个数据覆盖 | 接收中断自动读取转存 |
| 定时器 | 只能靠Delay函数实现定时 | 定时中断自动触发，不阻塞主程序 |

**核心价值**: 极大提高程序效率，实现**并行处理**的效果。

### 2.3 中断执行流程

```
主程序执行 → 中断条件满足 → 硬件自动跳转
                ↓
         保护现场（保存当前状态）
                ↓
         执行中断服务程序(ISR)
                ↓
         恢复现场（还原之前状态）
                ↓
         返回断点继续执行
```

**关键点**:
- **断点**: 主程序被暂停的位置
- **现场保护/恢复**: C语言中编译器自动完成
- **硬件自动跳转**: 无需手动调用中断函数

### 2.4 中断嵌套

**定义**: 当一个中断程序正在运行时，又有新的**更高优先级**的中断申请，CPU再次暂停当前中断程序，转而去处理新的中断。

**流程**:
```
主程序 → 中断A(低优先级) → 中断B(高优先级) → 中断A → 主程序
```

**条件**: 由高优先级的**抢占优先级**决定是否可嵌套。

## 三、STM32 中断系统概览

### 3.1 中断资源

STM32F1系列具有以下中断特性：

| 特性 | 说明 |
|------|------|
| 可屏蔽中断通道 | 68个（F1系列最多，具体型号可能不同） |
| 管理单元 | NVIC (Nested Vectored Interrupt Controller) |
| 优先级等级 | 16个可编程优先级 |
| 优先级分组 | 可设置抢占优先级和响应优先级 |

**支持中断的外设**:
- TIM定时器
- ADC模数转换器
- USART串口
- SPI通信
- I2C通信
- RTC实时时钟
- EXTI外部中断
- 等等...

### 3.2 中断类型

```
STM32中断资源
├── 内核中断（灰色标记）
│   ├── 复位中断 (Reset)
│   ├── NMI不可屏蔽中断
│   ├── 硬件失效 (HardFault)
│   ├── 存储管理错误
│   └── ...（一般用不到）
│
└── 外设中断
    ├── 窗口看门狗 (WWDG)
    ├── PVD电源电压监测
    ├── EXTI0~EXTI4 (独立通道)
    ├── EXTI9_5 (共用通道)
    ├── EXTI15_10 (共用通道)
    ├── TIM定时器
    ├── ADC
    ├── USART
    └── ...
```

### 3.3 中断向量表

**作用**: 解决硬件跳转固定地址与中断函数地址不固定的矛盾。

**原理**:
1. 内存中定义固定地址列表（中断向量表）
2. 中断发生时硬件跳到固定位置
3. 固定位置存放跳转指令，跳转到实际中断函数

**注意**: C语言编程时编译器自动处理，无需手动配置。

## 四、NVIC (嵌套中断向量控制器)

### 4.1 NVIC的作用

NVIC是**内核外设**，相当于CPU的"叫号系统"：

| 功能 | 说明 |
|------|------|
| 统一管理中断 | 接收所有外设的中断请求 |
| 优先级裁决 | 根据优先级决定中断先后顺序 |
| 单一输出 | 每次只向CPU输出一个中断请求 |

**比喻**: 医生(CPU)专心看病，NVIC负责排号叫号，按优先级安排病人。

### 4.2 中断优先级

#### 两种优先级类型

| 类型 | 功能 | 比喻 |
|------|------|------|
| **抢占优先级** | 可中断嵌套，高优先级可打断低优先级 | 急诊病人直接冲进诊室 |
| **响应优先级** | 仅影响排队顺序，不可嵌套 | 普通病人插队提前看病 |

**规则**:
1. 抢占优先级高的可以**中断嵌套**（打断正在执行的中断）
2. 响应优先级高的可以**优先排队**（同级别中先执行）
3. 两者都相同时，按**中断号**（向量表序号）排队

### 4.3 优先级分组

**原理**: 4位优先级寄存器，可切分为抢占优先级位和响应优先级位。

| 分组 | 抢占优先级位数 | 响应优先级位数 | 抢占优先级范围 | 响应优先级范围 |
|:----:|:-------------:|:-------------:|:-------------:|:-------------:|
| 组0 | 0位 | 4位 | 0 | 0~15 |
| 组1 | 1位 | 3位 | 0~1 | 0~7 |
| 组2 | 2位 | 2位 | 0~3 | 0~3 |
| 组3 | 3位 | 1位 | 0~7 | 0~1 |
| 组4 | 4位 | 0位 | 0~15 | 0 |

**注意**:
- 数值越小，优先级越高
- 0是最高优先级
- 分组方式在程序中自行选择配置

**配置示例**:
```c
// 选择分组2：2位抢占优先级 + 2位响应优先级
NVIC_PriorityGroupConfig(NVIC_PriorityGroup_2);
```

## 五、外部中断 (EXTI)

### 5.1 EXTI基本概念

**全称**: External Interrupt（外部中断）

**功能**: 监测指定GPIO口的电平变化，当电平变化时立即向NVIC发出中断申请。

**触发方式**:
| 方式 | 说明 |
|------|------|
| 上升沿 | 低电平变高电平的瞬间触发 |
| 下降沿 | 高电平变低电平的瞬间触发 |
| 双边沿 | 上升沿和下降沿都触发 |
| 软件触发 | 程序中执行代码触发 |

### 5.2 EXTI支持的GPIO口

**规则**:
- 支持**所有GPIO口**（PA、PB、PC、PD、PE、PF、PG）
- **相同Pin号**不能同时触发中断（如PA0和PB0只能选一个）
- 不同Pin号可同时使用（如PA6、PA7、PB0、PB1）

**原因**: AFIO数据选择器决定了相同Pin号只能有一个连接到EXTI通道。

### 5.3 EXTI通道分配

| 通道 | 来源 | 说明 |
|:----:|------|------|
| EXTI0~EXTI4 | GPIO Pin0~Pin4 | 独立通道，各自有独立中断函数 |
| EXTI9_5 | GPIO Pin5~Pin9 | 共用通道，共享一个中断函数 |
| EXTI15_10 | GPIO Pin10~Pin15 | 共用通道，共享一个中断函数 |
| 其他 | PVD、RTC闹钟、USB唤醒、以太网唤醒 | 蹭网通道（用于低功耗唤醒） |

**注意**: EXTI9_5和EXTI15_10共用中断函数，需要在函数内部根据标志位判断具体是哪个中断源。

### 5.4 EXTI触发响应方式

| 方式 | 说明 |
|------|------|
| **中断响应** | 正常流程，触发CPU执行中断函数 |
| **事件响应** | 不触发中断，而是触发其他外设（如ADC转换、DMA） |

### 5.5 EXTI内部结构详解

```
                    ┌─────────────────────────────────────┐
                    │           EXTI 内部结构              │
┌─────────┐        │                                     │
│ GPIOA   │──PA0───┤  ┌─────────┐                         │
│ GPIOB   │──PB0───┼──┤  AFIO   │──EXTI0──┐              │
│ GPIOC   │──PC0───┤  │ 选择器  │         │              │
│ ...     │        │  └─────────┘         │              │
└─────────┘        │                       ↓              │
                    │         ┌───────────────────────┐  │
                    │         │   边沿检测电路         │  │
                    │         │  ┌───────┐ ┌───────┐   │  │
                    │         │  │上升沿 │ │下降沿 │   │  │
                    │         │  │检测   │ │检测   │   │  │
                    │         │  └───┬───┘ └───┬───┘   │  │
                    │         └──────┼───────┼─────────┘  │
                    │                └───┬───┘            │
                    │                    ↓                │
                    │            ┌───────────────┐        │
                    │            │     或门      │        │
                    │            │  (触发信号)  │        │
                    │            └───────┬───────┘        │
                    │                    │                │
                    │        ┌──────────┴──────────┐       │
                    │        ↓                     ↓       │
                    │   ┌──────────┐          ┌──────────┐│
                    │   │ 中断通道  │          │ 事件通道  ││
                    │   │  挂起寄存器│          │ 脉冲发生  ││
                    │   │  屏蔽寄存器│          │   器     ││
                    │   └────┬─────┘          └────┬─────┘│
                    │        ↓                     ↓       │
                    │      NVIC                其他外设    │
                    │                        (ADC/DMA等)  │
                    └─────────────────────────────────────┘
```

**关键部件**:

| 部件 | 功能 |
|------|------|
| **AFIO数据选择器** | 从多个GPIO口中选择一个连接到EXTI通道 |
| **边沿检测电路** | 检测上升沿/下降沿/双边沿 |
| **或门** | 硬件触发或软件触发任一满足即可输出 |
| **挂起寄存器** | 中断标志位，可读取判断中断源 |
| **屏蔽寄存器** | 中断开关控制（1=允许，0=屏蔽） |
| **脉冲发生器** | 产生脉冲信号触发其他外设 |

### 5.6 AFIO (复用功能IO)

**功能**:
1. **复用功能引脚重映射**: 改变默认复用功能引脚的位置
2. **中断引脚选择**: 选择哪个GPIO口连接到EXTI通道

**寄存器**: `AFIO_EXTICR1~4` 用于配置外部中断引脚选择。

## 六、旋转编码器原理

### 6.1 编码器类型

| 类型 | 特点 | 应用 |
|------|------|------|
| 机械触点式 | 金属触点通断，带按键 | 音量调节、菜单选择 |
| 霍尔传感器式 | 非接触，磁感应 | 电机测速 |
| 光栅式 | 光电感应，精度高 | 高精度位置检测 |

### 6.2 编码器工作原理

#### 单向输出编码器
- 一路方波信号
- 可测**位置和速度**
- 不可测**方向**

#### 正交输出编码器（AB相）
- A相、B相两路方波信号
- **相位差90°**（正交波形）
- 可测**位置、速度、方向**

**方向判断**:
```
正转: B相滞后A相 90°
      A: ▁▂▄▆█▆▄▂▁
      B: ▁▁▂▄▆█▆▄▂

反转: B相超前A相 90°
      A: ▁▂▄▆█▆▄▂▁
      B: ▂▄▆█▆▄▂▁▁
```

### 6.3 旋转编码器电路

```
        VCC
         │
         R1 (10K上拉)
         │
触点 ────┼──── A端口输出
         │
        GND
         
         VCC
         │
         R2 (10K上拉)
         │
触点 ────┼──── B端口输出
         │
        GND
```

**说明**:
- 上拉电阻: 默认输出高电平
- 触点导通: 输出被拉低到GND
- 滤波电容: 防止信号抖动

## 七、外部中断适用场景

### 7.1 推荐使用外部中断的情况

| 场景 | 原因 |
|------|------|
| **旋转编码器** | 信号突发、快速，需立即响应 |
| **红外遥控接收** | 波形转瞬即逝，错过即丢失 |
| **高速脉冲计数** | 主程序轮询可能漏检 |

### 7.2 不推荐的情况

| 场景 | 原因 | 替代方案 |
|------|------|---------|
| **按键检测** | 抖动处理麻烦，信号非瞬态 | 主循环轮询或定时器中断 |

**原因**: 按键按下时间长（几十毫秒），不需要极快响应，且需处理抖动和松手检测，主循环或定时器中断更合适。

## 八、代码实现

### 8.1 红外对射传感器计次

```c
#include "stm32f10x.h"

volatile uint16_t Count = 0;

// 外部中断服务函数
void EXTI0_IRQHandler(void)
{
    if(EXTI_GetITStatus(EXTI_Line0) != RESET)
    {
        Count++;  // 计数加1
        
        // 清除中断标志位（必须！）
        EXTI_ClearITPendingBit(EXTI_Line0);
    }
}

void EXTI_Init_Config(void)
{
    // 1. 开启时钟：GPIOB 和 AFIO
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB | RCC_APB2Periph_AFIO, ENABLE);
    
    // 2. 配置GPIO为浮空输入
    GPIO_InitTypeDef GPIO_InitStruct;
    GPIO_InitStruct.GPIO_Pin = GPIO_Pin_0;
    GPIO_InitStruct.GPIO_Mode = GPIO_Mode_IN_FLOATING;
    GPIO_Init(GPIOB, &GPIO_InitStruct);
    
    // 3. 配置AFIO：选择GPIOB Pin0连接到EXTI0
    GPIO_EXTILineConfig(GPIO_PortSourceGPIOB, GPIO_PinSource0);
    
    // 4. 配置EXTI
    EXTI_InitTypeDef EXTI_InitStruct;
    EXTI_InitStruct.EXTI_Line = EXTI_Line0;          // EXTI0线路
    EXTI_InitStruct.EXTI_Mode = EXTI_Mode_Interrupt; // 中断模式
    EXTI_InitStruct.EXTI_Trigger = EXTI_Trigger_Rising; // 上升沿触发
    EXTI_InitStruct.EXTI_LineCmd = ENABLE;           // 使能
    EXTI_Init(&EXTI_InitStruct);
    
    // 5. 配置NVIC
    NVIC_InitTypeDef NVIC_InitStruct;
    NVIC_PriorityGroupConfig(NVIC_PriorityGroup_2);  // 优先级分组2
    
    NVIC_InitStruct.NVIC_IRQChannel = EXTI0_IRQn;    // EXTI0通道
    NVIC_InitStruct.NVIC_IRQChannelPreemptionPriority = 1; // 抢占优先级1
    NVIC_InitStruct.NVIC_IRQChannelSubPriority = 1;      // 响应优先级1
    NVIC_InitStruct.NVIC_IRQChannelCmd = ENABLE;
    NVIC_Init(&NVIC_InitStruct);
}

int main(void)
{
    // OLED初始化等...
    
    EXTI_Init_Config();
    
    while(1)
    {
        // 显示计数值
        OLED_ShowNumber(0, 0, Count, 5);
    }
}
```

### 8.2 旋转编码器计次

```c
#include "stm32f10x.h"

volatile int16_t Encoder_Count = 0;

// EXTI0中断服务函数 (A相)
void EXTI0_IRQHandler(void)
{
    if(EXTI_GetITStatus(EXTI_Line0) != RESET)
    {
        // 读取B相电平判断方向
        if(GPIO_ReadInputDataBit(GPIOB, GPIO_Pin_1) == 0)
        {
            Encoder_Count++;  // 正转
        }
        else
        {
            Encoder_Count--;  // 反转
        }
        
        EXTI_ClearITPendingBit(EXTI_Line0);
    }
}

// EXTI1中断服务函数 (B相)
void EXTI1_IRQHandler(void)
{
    if(EXTI_GetITStatus(EXTI_Line1) != RESET)
    {
        // 读取A相电平判断方向
        if(GPIO_ReadInputDataBit(GPIOB, GPIO_Pin_0) == 0)
        {
            Encoder_Count--;  // 反转
        }
        else
        {
            Encoder_Count++;  // 正转
        }
        
        EXTI_ClearITPendingBit(EXTI_Line1);
    }
}

void Encoder_Init(void)
{
    // 开启时钟
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB | RCC_APB2Periph_AFIO, ENABLE);
    
    // 配置GPIO: PB0(A相), PB1(B相) 为浮空输入
    GPIO_InitTypeDef GPIO_InitStruct;
    GPIO_InitStruct.GPIO_Pin = GPIO_Pin_0 | GPIO_Pin_1;
    GPIO_InitStruct.GPIO_Mode = GPIO_Mode_IN_FLOATING;
    GPIO_Init(GPIOB, &GPIO_InitStruct);
    
    // AFIO配置
    GPIO_EXTILineConfig(GPIO_PortSourceGPIOB, GPIO_PinSource0); // A相→EXTI0
    GPIO_EXTILineConfig(GPIO_PortSourceGPIOB, GPIO_PinSource1); // B相→EXTI1
    
    // EXTI配置：双边沿触发
    EXTI_InitTypeDef EXTI_InitStruct;
    EXTI_InitStruct.EXTI_Line = EXTI_Line0 | EXTI_Line1;
    EXTI_InitStruct.EXTI_Mode = EXTI_Mode_Interrupt;
    EXTI_InitStruct.EXTI_Trigger = EXTI_Trigger_Rising_Falling; // 双边沿
    EXTI_InitStruct.EXTI_LineCmd = ENABLE;
    EXTI_Init(&EXTI_InitStruct);
    
    // NVIC配置
    NVIC_PriorityGroupConfig(NVIC_PriorityGroup_2);
    
    NVIC_InitTypeDef NVIC_InitStruct;
    // EXTI0配置
    NVIC_InitStruct.NVIC_IRQChannel = EXTI0_IRQn;
    NVIC_InitStruct.NVIC_IRQChannelPreemptionPriority = 1;
    NVIC_InitStruct.NVIC_IRQChannelSubPriority = 1;
    NVIC_InitStruct.NVIC_IRQChannelCmd = ENABLE;
    NVIC_Init(&NVIC_InitStruct);
    
    // EXTI1配置
    NVIC_InitStruct.NVIC_IRQChannel = EXTI1_IRQn;
    NVIC_Init(&NVIC_InitStruct);
}

int main(void)
{
    // 初始化...
    Encoder_Init();
    
    while(1)
    {
        OLED_ShowSignedNumber(0, 0, Encoder_Count, 5);
    }
}
```

### 10.1 中断系统核心要点

```
┌─────────────────────────────────────────────────────┐
│                    中断系统架构                      │
├─────────────────────────────────────────────────────┤

│                         └────┬────┘          └────────┘│

└─────────────────────────────────────────────────────┘
```

### 10.2 配置流程口诀

```
外部中断配置步骤：
1. 开时钟  (RCC: GPIO + AFIO)
2. 配GPIO  (浮空/上拉/下拉输入)
3. 配AFIO  (GPIO_EXTILineConfig)
4. 配EXTI  (线路、模式、触发方式)
5. 配NVIC  (优先级分组、优先级、使能)
6. 写ISR   (中断服务函数，记得清标志位！)
```

### 10.3 易错点提醒

| 易错点 | 正确做法 |
|--------|---------|
| 忘记清除中断标志位 | `EXTI_ClearITPendingBit()` |
| 相同Pin号重复配置 | PA0和PB0只能选一个 |
| 共用通道不判断来源 | EXTI9_5/EXTI15_10需判断挂起寄存器 |
| 优先级超出范围 | 根据分组确认抢占/响应优先级范围 |
